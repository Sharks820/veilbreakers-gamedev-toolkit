    **Version-aware BSDF socket lookup (CRITICAL):**
    ```python
    BSDF_INPUT_MAP = {
        "base_color": "Base Color",
        "metallic": "Metallic",
        "roughness": "Roughness",
        "normal": "Normal",
        "ior": "IOR",
        "alpha": "Alpha",
        # Blender 4.0+ renamed sockets (use new names -- old names raise KeyError):
        "subsurface": "Subsurface Weight",
        "specular": "Specular IOR Level",
        "transmission": "Transmission Weight",
        "coat": "Coat Weight",
        "sheen": "Sheen Weight",

    }
    ```

    Also create `_get_bsdf_input(bsdf_node, semantic_name)` helper that tries the BSDF_INPUT_MAP name first, then falls back to common pre-4.0 names, returning the socket or raising a clear error.

    **PBR channel config:**
    Create `_build_channel_config()` returning a dict mapping channel names to (file_suffix, bsdf_input_name, colorspace, needs_normal_node) tuples. Channels: albedo ("_albedo", "Base Color", "sRGB", False), metallic ("_metallic", "Metallic", "Non-Color", False), roughness ("_roughness", "Roughness", "Non-Color", False), normal ("_normal", "Normal", "Non-Color", True), ao ("_ao", None, "Non-Color", False -- AO mixed via MixRGB or connected to a custom group).

    **1. handle_create_pbr_material(params: dict) -> dict** (TEX-01)
    - Accept: `name` (str), `texture_dir` (str, optional -- directory with texture files), `texture_size` (int, default 1024), `object_name` (str, optional -- assign material after creation)
    - Create new material with `use_nodes=True`
    - Clear default nodes, create ShaderNodeOutputMaterial and ShaderNodeBsdfPrincipled
    - Link BSDF output to Output Surface input
    - For each channel in `_build_channel_config()`:
      - Create ShaderNodeTexImage node, position at staggered x/y
      - If `texture_dir` provided, search for `{name}{suffix}.{png|jpg|tga|exr}` and load via `bpy.data.images.load(filepath)`
      - If no file found or no `texture_dir`, create blank image via `bpy.data.images.new(name+suffix, texture_size, texture_size)`
      - Set `image.colorspace_settings.name = colorspace`
      - For normal: insert ShaderNodeNormalMap between tex and BSDF Normal input
      - For AO: create ShaderNodeMixRGB (Multiply), connect AO to Color2 and Base Color to Color1, output to BSDF Base Color (replacing direct albedo link)
      - For others: link tex Color output to BSDF input via `_get_bsdf_input()`
    - If `object_name` provided, assign material to first material slot
    - Return: `{material_name, channels: {channel: node_name}, texture_size, assigned_to}`

    **2. handle_bake_textures(params: dict) -> dict** (TEX-07)
    - Accept: `object_name`, `bake_type` (NORMAL|AO|COMBINED|ROUGHNESS|EMIT|DIFFUSE), `source_object` (optional, for selected-to-active), `image_name` (str -- which image texture node to bake into), `margin` (int, default 16), `cage_extrusion` (float, default 0.1), `samples` (int, default 32)
    - Validate bake_type with `_validate_bake_params()`
    - CRITICAL: Store `prev_engine = bpy.context.scene.render.engine`, set to `'CYCLES'`, set `bpy.context.scene.cycles.samples = samples`
    - If `source_object` (selected-to-active bake): deselect all, select source (high-poly), select target (low-poly), set target as active object
    - If no `source_object` (self-bake): deselect all, select and activate target
    - Find the target image texture node by matching `node.image.name == image_name`, set `node.select = True` and `mat.node_tree.nodes.active = node`
    - Use `get_3d_context_override()` + `bpy.context.temp_override()` for the bake call
    - Set `normal_space='TANGENT'` when bake_type is NORMAL
    - ALWAYS restore previous render engine in a `finally` block
    - Return: `{baked: True, bake_type, object, source, image}`

    **3. handle_validate_texture(params: dict) -> dict** (TEX-10)
    - Accept: `object_name`, `texture_size` (int, for UV coverage check)
    - For each material on the object, inspect all Image Texture nodes
    - For each image: check resolution (power-of-two), format, color space, bit depth
    - Use `_validate_texture_metadata(width, height, format_name, colorspace)` pure function
    - Check UV coverage: if object has UV layer, compute percentage of UV space covered by UV islands (approximate via sampling or bmesh UV area sum)
    - Return: `{object_name, textures: [{name, width, height, format, colorspace, is_power_of_two, issues: [...]}], uv_coverage_pct, overall_valid}`

    **Testing pattern:** Follow Phase 2 pattern -- extract pure-logic functions (`_build_channel_config`, `_validate_texture_metadata`, `_validate_bake_params`, `BSDF_INPUT_MAP`) that are testable without bpy. Test these in `test_texture_handlers.py` using the existing conftest.py stubs.
