**CRITICAL DESIGN PRINCIPLE:** These generators produce WIRING code -- MonoBehaviours that call into existing static utility classes. **Generator 1: generate_player_combat_script** (VB-01)
Parameters: `light_combo_count=3, heavy_combo_count=2, dodge_iframe_duration=0.2, dodge_distance=4.0, block_stamina_drain=10.0, stamina_max=100.0, stamina_regen_rate=15.0, namespace="VeilBreakers.Combat"`
Generates C# with:
- `VB_PlayerCombat` MonoBehaviour with FSM states: Idle, LightAttack, HeavyAttack, Dodge, Block, HitReaction, Dead
- `CombatState` enum for state machine
- Light attack combo chain: N hits (configurable), each advances on input within timing window (0.5s)
- Heavy attack combo chain: M hits with longer wind-up and higher damage multiplier
- Dodge with i-frames: disable hitbox for configurable duration, move character dodge_distance in input direction
- Block: reduce incoming damage by 50%, drain stamina per blocked hit, break block if stamina depleted
- Hit reaction: stagger animation trigger, recovery time
- Stamina system: consumed by dodge/block/heavy attack, regenerates passively when not attacking (configurable rate), EventBus.StaminaChanged
- Combo reset on timeout (configurable window between hits)
- Integration: GetComponent<Combatant>() for stats, DamageCalculator.Calculate() for damage, EventBus.DamageDealt on hit
- Animation hooks: Animator.SetTrigger for each state
- Input hooks: virtual methods for GetAttackInput(), GetDodgeInput(), GetBlockInput()
- MUST NOT contain `using UnityEditor`

**Generator 2: generate_ability_system_script** (VB-02)
Parameters: `max_ability_slots=4, mana_max=100.0, mana_regen_rate=5.0, namespace="VeilBreakers.Combat"`

- `AbilitySlot` class: ability reference, cooldown timer, brand requirement, mana cost, activation key
- Ability slots array (configurable max, default 4)
- Brand-specific ability activation: check if player's current brand matches ability requirement
- Cooldown management: per-ability cooldown timers, UI-friendly cooldown percentage
- Mana/stamina resource: shared or separate pools (configurable), consumption on ability use, passive regen
- Ability activation: validate cooldown + resource + brand, fire ability, start cooldown, deduct resource
- EventBus.AbilityUsed integration
- ScriptableObject-based ability definitions (reference existing AbilityData SO pattern)
- MUST NOT contain `using UnityEditor`

- Combo trigger tracking: consecutive hits of same synergy tier build combo meter

- Visual feedback hooks: color tinting per tier (gold=FULL, blue=PARTIAL, gray=NEUTRAL, red=ANTI)

- MUST NOT contain `using UnityEditor`
- MUST contain `SynergySystem.GetSynergyTier` (proof of delegation)
- MUST contain `SynergySystem.GetDamageBonus` (proof of delegation)

- Threshold triggers: at 25% (Tainted), 50% (Corrupted), 75% (Consumed), 100% (max)

- NPC reaction changes: flag for NPC dialogue system to check corruption state
- Visual feedback: corruption percentage drives VFX intensity (shader parameter _CorruptionLevel)
- EventBus.CorruptionChanged integration
- Does NOT contain corruption formula -- delegates entirely to CorruptionSystem static class
- MUST NOT contain `using UnityEditor`
- MUST contain `CorruptionSystem.GetStatMultiplier` (proof of delegation)
- MUST contain `CorruptionSystem.GetCorruptionState` (proof of delegation)
