From unity_server.py existing compound tool wiring pattern:
```python
@mcp.tool()
async def unity_gameplay(
    action: Literal["create_mob_controller", ...],
    name: str = "default",
    ...
) -> str:
    """Docstring with all actions documented."""
    try:
        if action == "create_mob_controller":
            return await _handle_gameplay_mob_controller(...)
        ...
    except Exception as e:
        return json.dumps({"error": str(e)})
```

Key wiring rules from established pattern:
1. Literal action parameter lists all actions
2. Tool docstring describes all actions with their parameters
3. Each action dispatches to corresponding template generator
4. For C# outputs: write via _write_to_unity to Assets/Scripts/Runtime/GameSystems/, return JSON with file_path + next_steps
5. For multi-file outputs (settings menu = C# + UXML + USS, input config = JSON + C#): write all files, return combined result
6. For non-C# outputs (InputActions JSON): write directly, not through _write_to_unity
7. All user-provided strings sanitized via _sanitize_cs_identifier before passing to generators
8. Runtime scripts go to Assets/Scripts/Runtime/ (NOT Assets/Editor/Generated/ which is for editor scripts)
