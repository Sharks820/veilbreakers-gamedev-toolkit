## Task 2: Implement `ring_for_position`, `weighted_building_type`, and road network organic

**What:** Implement the concentric ring district assignment, soft gradient building type selection, and L-system road organics.

**Actions:**

1. Implement `ring_for_position(pos, center, radius)` in `_settlement_grammar.py`:
   - Use `RING_THRESHOLDS` list, iterate until `dist_norm < threshold`
   - `dist_norm = Vector2D(pos - center).length / radius`
   - Return the ring name string
   - Handle edge case: dist_norm > 1.0 → return "outskirts"

2. Implement `weighted_building_type(district, neighbor_district, dist_to_boundary, seed)`:
   - `rng = random.Random(seed)`
   - If `dist_to_boundary < 0.05` (within 5% of zone radius at boundary): 30% chance to pull from neighbor_district types
   - Otherwise: pull from own district types only
   - `return rng.choice(selected_list)`

3. Implement `perturb_road_points(start, end, rng, amplitude=1.5, steps=3)`:
   - Insert `steps-1` intermediate points between start and end
   - For each intermediate: lerp along segment, then offset perpendicular by `rng.uniform(-amplitude, amplitude)`
   - Return list of tuples: `[start, *midpoints, end]`
   - Perpendicular: if segment direction is `(dx, dy)`, perpendicular is `(-dy, dx)` normalized

4. Implement `generate_road_network_organic(center, radius, seed, settlement_points)`:
   - `rng = random.Random(seed)`
   - Build MST edges from settlement_points using Kruskal's (replicate `compute_mst_edges` logic from road_network.py — do NOT import road_network.py to keep grammar pure)
   - For each MST edge: classify as "main_road" if total_degree >= 3 (intersection), else "alley"
   - For main roads: apply `perturb_road_points` with amplitude=1.5, steps=4
   - For alleys: apply `perturb_road_points` with amplitude=0.8, steps=3
   - Add L-system branch alleys: for each main road midpoint, 40% chance to emit a branching alley toward nearest building cluster
   - Return list of segment dicts: `{"points": [...], "width": float, "style": "main_road"|"alley", "seed": int}`
   - Main road width: 4.0m; alley width: 2.0m (D-02)

5. Run: `pytest tests/test_settlement_grammar.py -v`
   - `test_ring_district_assignment` should now PASS
   - `test_weighted_building_type_returns_valid_type` should now PASS
   - Road tests are added in next step

**Verification:** All ring and building type tests pass. Manual sanity: call `generate_road_network_organic` with 5 settlement points and verify segment count > 4 (at least MST edges).

## Task 3: Implement OBB lot subdivision and building assignment

**What:** Implement the recursive OBB lot subdivision algorithm and building placement onto lots.

**Actions:**

1. Implement `_block_area(polygon_verts)` helper (private):
   - Shoelace formula: `0.5 * abs(sum(x_i*(y_{i+1} - y_{i-1})))`

2. Implement `_obb_split_polygon(polygon_verts, rng, split_ratio_range=(0.4, 0.6))` helper (private):
   - `pts = np.array(polygon_verts)`
   - Compute centroid, center points
   - PCA via `np.cov` + `np.linalg.eigh` — get principal axis
   - Project all points onto principal axis, find min/max
   - Choose split point at `rng.uniform(*split_ratio_range)` along the range
   - Clip polygon at split line → returns two sub-polygons
   - If clip fails (degenerate polygon): return `(polygon_verts, None)`

3. Implement `subdivide_block_to_lots(block_polygon, district, seed, min_lot_area=25.0)`:
   - `rng = random.Random(seed)`
   - Base case: if `_block_area(block_polygon) < min_lot_area * 2`, return `[{"polygon": block_polygon, "street_frontage_edge": _nearest_edge_to_centroid(block_polygon)}]`
   - Recursive case: split via `_obb_split_polygon`, recurse on both halves with `seed+1` and `seed+2`
   - Each returned lot dict: `{"polygon": verts, "street_frontage_edge": (v1, v2), "district": district, "area": float}`
   - Lot size by district: market=large (min_lot_area=100), civic=80, residential=25, industrial=60, outskirts=150

4. Implement `assign_buildings_to_lots(lots, center, radius, veil_pressure, seed)`:
   - `rng = random.Random(seed)`
   - For each lot:
     - Compute lot centroid
     - Determine district via `ring_for_position`
     - Determine neighbor district (find which ring boundary is nearest)
     - Compute `dist_to_boundary` (distance to nearest ring boundary)
     - Get `building_type = weighted_building_type(district, neighbor_district, dist_to_boundary, rng.randint(0, 99999))`
     - Get fill rate from `DISTRICT_FILL_RATES[district]`
     - If `rng.random() > fill_rate`: lot becomes open space → `building_type = None`
     - Return lot dict extended with: `{"building_type": building_type|None, "orientation_edge": street_frontage_edge}`

5. Add tests to `test_settlement_grammar.py`:
   - `test_obb_lot_subdivision_basic`: provide a 20x20m square polygon, assert at least 2 lots returned
   - `test_obb_lot_subdivision_respects_min_area`: provide 8x8m polygon (< 2 * min_area), assert 1 lot returned
   - `test_assign_buildings_respects_fill_rate`: run 100 trials for "residential" district, assert ~80% have building_type (not None)
   - `test_assign_buildings_lot_has_frontage`: assert every lot has `orientation_edge` set

6. Run: `pytest tests/test_settlement_grammar.py -v`

**Verification:** All lot subdivision and building assignment tests pass. No numpy import errors.

## Task 4: Curb geometry in road_network.py + prop manifest grammar

**What:** Extend `_road_segment_mesh_spec()` with curb/gutter cross-section geometry, and implement `generate_prop_manifest()` in the grammar.

**Actions:**

1. Read `Tools/mcp-toolkit/blender_addon/handlers/road_network.py` (full file, existing `_road_segment_mesh_spec` function).

2. In `road_network.py`, create `_road_segment_mesh_spec_with_curbs(start, end, width, curb_height=0.15, gutter_width=0.3)`:
   - Cross-section has 6 vertex columns (per point along segment):
     - col 0: outer-left gutter (X = -width/2 - gutter_width, Z = 0)
     - col 1: curb-top-left (X = -width/2, Z = curb_height)
     - col 2: inner-left gutter (X = -width/2 + gutter_width*0.3, Z = 0)
     - col 3: inner-right gutter (X = width/2 - gutter_width*0.3, Z = 0)
     - col 4: curb-top-right (X = width/2, Z = curb_height)
     - col 5: outer-right gutter (X = width/2 + gutter_width, Z = 0)
   - Generate vertex grid: for each point on segment centerline, emit all 6 cross-section verts
   - Faces: quad strips between adjacent cross-section columns
   - Return: `{"vertices": [...], "faces": [...], "uv_layers": {"road_surface": [...], "curb": [...]}}`
   - UV tiling: road_surface UV = (u_along_road, v_across_road), cobblestone tiles every 1m → `u = dist_along / 1.0`
   - Curb UV: tiles every 0.5m along curb edge

3. Update `compute_road_network()` in `road_network.py` to call `_road_segment_mesh_spec_with_curbs` instead of `_road_segment_mesh_spec` for segments with style `"main_road"` or `"cobblestone"`. Keep old function for trail/path styles.

4. In `_settlement_grammar.py`, implement `prop_tier_for_pressure(pressure)`:
   - Iterate `CORRUPTION_TIERS`, return first matching `(band, spacing_min, spacing_max)`

5. In `_settlement_grammar.py`, implement `generate_prop_manifest(road_segments, center, radius, veil_pressure, seed)`:
   - `rng = random.Random(seed)`
   - Get `band, spacing_min, spacing_max = prop_tier_for_pressure(veil_pressure)`
   - For each road segment:
     - Sample prop positions along road using Poisson-disk-like spacing: start at `rng.uniform(spacing_min, spacing_max)` from segment start, advance by `rng.uniform(spacing_min, spacing_max)` each step
     - For each position along road edge (both sides, offset by `width/2 + 0.3m`):
       - Determine district via `ring_for_position`
       - Select prop type from `ROAD_PROP_TYPES[district]` (new constant below)
       - Emit: `{"prop_type": str, "position": (x,y,z), "rotation_z": float, "corruption_band": band, "cache_key": (prop_type, band)}`
   - Add constant `ROAD_PROP_TYPES`:
     ```python

         "civic_ring":    ["lantern_post", "bench", "notice_board"],
         "residential":   ["lantern_post", "trough", "bench"],
         "industrial":    ["barrel_cluster", "cart", "trough"],
         "outskirts":     ["lantern_post"],
     ```
