# Phase 36-02: World Composer — Tripo Prop Pipeline and Blender Wiring

**Phase:** 36 — World Composer
**Plan:** 02 of 02
**Requirement:** MESH-08 (market stalls, wells, lamp posts — AAA quality, Tripo AI sourced)
**Depends on:** Phase 36-01 (grammar layer complete, prop_manifest in settlement data), Phase 35 (asset_pipeline Tripo + texture pipeline)
**Status:** Ready for execution (after 36-01 complete)

## Goal

Wire the prop manifest from Plan 01's grammar layer into actual 3D objects in Blender. Every street prop is generated via Tripo AI using dark fantasy prompts, cached by (type, corruption_band), post-processed through the Phase 35 texture pipeline, and placed on terrain with correct normal alignment. The final `handle_generate_settlement()` call produces a complete, walkable medieval town with road geometry including curbs and AAA quality props.

## Deliverables

- `worldbuilding.py` — extended `handle_generate_settlement()` with Tripo prop pipeline + curb road materialization
- `settlement_generator.py` — `generate_settlement()` return includes `prop_manifest` from Plan 01
- `tests/test_settlement_generator.py` — new integration tests for prop manifest structure and materialization

## Task 2: Pre-generate props for a town (Tripo batch pass)

**What:** Before materializing a settlement, generate the unique (prop_type, corruption_band) combinations needed for the town via Tripo. This separates slow Tripo calls from fast Blender object placement.

**Actions:**

1. In `worldbuilding.py`, add function `prefetch_town_props(prop_manifest, veil_pressure)`:
   - Extract unique `cache_key` values from prop_manifest (set of (type, band) tuples)
   - For each unique key not already in `_PROP_CACHE`:
     - Get prompt via `get_prop_prompt(prop_type, corruption_band)`
     - Call `_get_or_generate_prop(prop_type, corruption_band, prompt, ...)`
     - Log: `f"Generated prop: {prop_type} ({corruption_band}) -> {glb_path}"`
   - Return dict: `{cache_key: glb_path}` for all resolved props
   - Log total count: `f"Prefetched {len(unique_keys)} prop types ({len(already_cached)} from cache)"`

2. Call `prefetch_town_props()` from `handle_generate_settlement()` BEFORE the prop placement loop:
   ```python
   # In handle_generate_settlement():
   prop_manifest = settlement_data.get("prop_manifest", [])
   if prop_manifest:
       prefetch_town_props(prop_manifest, veil_pressure)
   ```

3. Add `post_process` flag to prop generation:
   - After each successful Tripo generate_3d call: call `asset_pipeline` action=`post_process_model` on the GLB
   - This runs the Phase 35 delight + validate pipeline
   - If post_process fails: log warning, use raw GLB (non-fatal)

4. Add a separate `blender_worldbuilding` action `prefetch_settlement_props` to `blender_server.py`:
   - Allows calling prefetch independently from materialization
   - Useful for pre-warming cache before a generation session
   - Returns: `{"prefetched": N, "from_cache": M, "failed": K, "prop_types": [...]}`

5. Add test: `test_prefetch_returns_summary_dict` — mock `_get_or_generate_prop`, assert return dict has correct keys.

**Verification:** `pytest tests/test_settlement_generator.py -k "prefetch" -v` passes. Manual check: call prefetch for a 3-prop manifest, verify 3 cache entries populated.

## Task 3: Materialize props in Blender with terrain alignment

**What:** Replace `_spawn_catalog_object()` calls for street props with GLB import + terrain normal alignment.

**Actions:**

1. Read `Tools/mcp-toolkit/blender_addon/handlers/worldbuilding.py` (the `_scatter_settlement_props` section and `_spawn_catalog_object` function).

2. In `worldbuilding.py`, add function `_materialize_prop(prop_spec, glb_path, terrain_object)`:
   - Import GLB: use `bpy.ops.import_scene.gltf(filepath=glb_path)`
   - Get imported objects: `imported = [o for o in bpy.context.selected_objects]`
   - Move to prop_spec position: `obj.location = Vector(prop_spec["position"])`
   - Apply rotation_z: `obj.rotation_euler.z = prop_spec["rotation_z"]`
   - Terrain normal alignment:
     - Raycast downward from position + (0,0,1) against terrain_object
     - If hit: adjust Z to hit point Z + 0.01 (snap to surface)
     - Compute surface normal from hit, align object to normal via `track_to` rotation math
   - Apply LOD: call existing Phase 30 LOD pipeline on imported objects
     - `blender_lod` action=`generate_lods` with preset="prop" (or "small_prop")
   - Parent to settlement collection

3. In `handle_generate_settlement()`, replace the old `_scatter_settlement_props()` materialization with:
   ```python
   for prop_spec in settlement_data.get("prop_manifest", []):
       glb_path = _PROP_CACHE.get(prop_spec["cache_key"])
       if glb_path:
           _materialize_prop(prop_spec, glb_path, terrain_obj)
       else:
           # Fallback: skip prop (no cube proxies per D-07)
           pass
   ```

4. Add prop collection organization:
   - Create Blender collection `{settlement_name}_Props`
   - Sub-collections per district: `{settlement_name}_Props_Market`, `_Props_Civic`, etc.
   - All materialized props go into the correct sub-collection

5. Add visual verification step at end of prop materialization:
