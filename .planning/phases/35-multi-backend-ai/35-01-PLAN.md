# Phase 35: Multi-backend AI Integration - Plan 01

**Created:** 2026-03-31
**Requirements:** MESH-12
**Depends on:** Phase 30 (mesh quality baseline — COMPLETE)

## Goal

Close the Tripo pipeline gap: extract PBR textures from GLB into standalone files, wire them into Blender's node tree, apply weathering overlay on top (not replacing), validate art style against VB palette rules, and score/auto-select the best of 4 variants. Zero blank textures after the pipeline runs.

## Success Criteria

1. A downloaded Tripo GLB produces standalone albedo/normal/orm PNG files on disk — confirmed by file existence assertions in tests
2. `cleanup_ai_model` on a Tripo import preserves Tripo's textures in the node tree — no blank gray material
3. Weathering vertex colors composite over the albedo texture via Mix Color node, not replacing it
4. Art style validation runs automatically per-variant and flags issues with deviation metrics
5. End-to-end pipeline from `generate_3d` to Unity-ready FBX completes with all 5 PBR channels wired

### Task 1: GLB Texture Extractor Module

**Files:**
- CREATE `Tools/mcp-toolkit/src/veilbreakers_mcp/shared/glb_texture_extractor.py`
- CREATE `Tools/mcp-toolkit/tests/test_glb_texture_extractor.py`

**What to build:**

`extract_glb_textures(glb_path: str, out_dir: str) -> dict[str, str]`

Returns a channel map: `{"albedo": path, "normal": path, "orm": path, "ao": path}`. Keys are only present if that channel exists in the GLB.

Algorithm:
1. Try `pygltflib` import. If unavailable, fall back to manual struct reader using the same pattern as `pipeline_runner._validate_glb`.
2. Load the binary blob (BIN chunk) from the GLB.
3. Parse `materials[0].pbrMetallicRoughness` for `baseColorTexture`, `metallicRoughnessTexture`.
4. Parse `materials[0].normalTexture`, `occlusionTexture`, `emissiveTexture`.
5. For each texture reference: resolve `textures[i].source` -> `images[j]` -> `bufferViews[k]`.
6. Slice `blob[byteOffset : byteOffset + byteLength]` for each image.
7. Detect MIME type from `images[j].mimeType` or magic bytes. Write `.png` or `.jpg`.
8. Return `{"albedo": path, "orm": path, "normal": path}` (ao if separate from orm, emissive if present).

Also provide `get_glb_texture_count(glb_path: str) -> int` — returns number of embedded images without extracting.

- `test_handles_missing_normal_gracefully` — GLB with only albedo, verify no KeyError
- `test_buffer_view_offset_alignment` — bufferView with non-zero byteOffset, verify correct bytes extracted
- `test_get_glb_texture_count` — returns correct count
- `test_fallback_when_pygltflib_absent` — mock ImportError, verify struct reader path runs

### Task 2: Tripo Post-Processor (delight + validate + score)

**Files:**
- CREATE `Tools/mcp-toolkit/src/veilbreakers_mcp/shared/tripo_post_processor.py`
- CREATE `Tools/mcp-toolkit/tests/test_tripo_post_processor.py`

**What to build:**

`async def post_process_tripo_model(glb_path: str, out_dir: str) -> dict`

Returns:
```python
{
    "channels": {"albedo": path, "orm": path, "normal": path},
    "albedo_delit": path_or_none,
    "palette_validation": {passed: bool, issues: [...], stats: {...}},
    "roughness_validation": {passed: bool, variance: float},
    "channel_score": 0-100,
    "texture_dir": path,
}
```

Steps:
1. Call `extract_glb_textures(glb_path, out_dir/textures)` — get channel map
2. If `"albedo"` in channels: call `delight_albedo(albedo_path, albedo_delit_path)` — save de-lit version
3. Validate palette against `validate_palette(albedo_delit or albedo)` — get issues + stats
4. Validate roughness via `validate_roughness_map(orm_path)` — checks G channel if ORM or direct roughness map
5. Score = 25 * has_albedo + 25 * has_orm + 25 * has_normal + 15 * palette_passed + 10 * roughness_passed

Also build `score_variants(post_results: list[dict]) -> list[dict]` — returns variants sorted by `channel_score` descending. Best variant first.

- `test_score_variants_orders_by_channel_score`
- `test_score_is_100_for_perfect_model` — all channels + passed validations
- `test_palette_deviation_metrics_in_output` — issues list contains `{rule, value, threshold}`

### Task 3: Blender Texture Wiring Handler

**Files:**
- MODIFY `Tools/mcp-toolkit/blender_addon/handlers/texture.py` — add `load_extracted_textures` handler
- MODIFY `Tools/mcp-toolkit/blender_addon/handlers/weathering.py` — add `mix_weathering_over_texture` handler
- MODIFY `Tools/mcp-toolkit/src/veilbreakers_mcp/blender_server.py` — register new commands in the `blender_texture` tool

**What to build in texture.py:**

`handle_load_extracted_textures(data: dict) -> dict`

Parameters: `object_name`, `albedo_path`, `normal_path` (optional), `orm_path` (optional), `albedo_delit_path` (optional — preferred over albedo if present).

Steps inside Blender:
1. Get `bpy.data.objects[object_name].active_material`. Create new material if none.
2. Enable `use_nodes = True`.
3. Get or create Principled BSDF node.
4. Load albedo_delit_path (or albedo_path) as sRGB image node -> wire to `Base Color`.
5. If orm_path: load as Non-Color image node -> Separate RGB node -> G to `Roughness`, B to `Metallic`, R to an `AO Mix` (MixColor Multiply before Base Color).
6. If normal_path: load as Non-Color -> Normal Map node -> wire to `Normal`.
7. Return `{status, channels_loaded: ["albedo", "orm", "normal"], warnings: [...]}`.

**What to build in weathering.py:**

`handle_mix_weathering_over_texture(data: dict) -> dict`

Parameters: `object_name`, `weathering_strength` (0.0-1.0, default 0.4).

Steps:
1. Get the object's material node tree.
2. Find or create a `Color Attribute` node reading the `"weathering"` vertex color layer.
3. Insert a `Mix Color` node (mode=Multiply, factor=weathering_strength) between the albedo texture node output and the BSDF Base Color input.
4. Wire: albedo_tex.Color -> MixColor.A; ColorAttribute.Color -> MixColor.B; MixColor.Result -> BSDF.Base Color.
5. Return `{status, mix_node_created: bool}`.

**Register new actions in blender_server.py:**
- Add `"load_extracted_textures"` and `"mix_weathering_over_texture"` to the `blender_texture` action Literal list.
- Add handler dispatch cases.

**What to modify in pipeline_runner.py:**

Add `has_extracted_textures: bool = False` parameter to `cleanup_ai_model`. When True:
- Skip step 7 (`texture_create_pbr`)
- Instead call `texture_load_extracted_textures` with the paths (add `texture_channels: dict | None = None` param)

```python
async def cleanup_ai_model(
    self,
    object_name: str,
    poly_budget: int = 50000,
    enhance_profile: str = "prop",
    skip_enhance: bool = False,
    has_extracted_textures: bool = False,
    texture_channels: dict | None = None,
) -> dict:
    ...
    # Step 7: Load textures (extracted) or create blank PBR
    if has_extracted_textures and texture_channels:
        load_result = await self.blender.send_command(
            "texture_load_extracted_textures",
            {"object_name": object_name, **texture_channels}
        )
        results["steps"]["load_textures"] = load_result
    else:
        pbr_result = await self.blender.send_command(
            "texture_create_pbr",
            {"name": object_name, "object_name": object_name},
        )
        results["steps"]["create_pbr"] = pbr_result
```

**What to modify in blender_server.py generate_3d handler:**

After `_poll_and_download_variants` returns:
1. For each verified model: call `await post_process_tripo_model(model["path"], model_out_dir)`.
2. Import the best-scored variant into Blender (or all variants in grid as before).
3. After import: call `cleanup_ai_model(..., has_extracted_textures=True, texture_channels=post_result["channels"])`.
4. After cleanup: call `blender.send_command("mix_weathering_over_texture", {"object_name": obj_name})`.
5. Include post-processing results (palette validation, scores) in the returned JSON.
6. Update `next_steps` to reflect the completed pipeline (not "pick variant manually").
