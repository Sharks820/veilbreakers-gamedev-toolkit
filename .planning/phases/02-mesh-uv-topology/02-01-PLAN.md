    Create `Tools/mcp-toolkit/blender_addon/handlers/mesh.py` with three handler functions:

    **1. handle_analyze_topology(params: dict) -> dict** (MESH-01)
    - Accept `object_name` param, resolve via `bpy.data.objects.get(name)`
    - Validate object exists and is MESH type
    - Create bmesh via `bm = bmesh.new(); bm.from_mesh(obj.data)`
    - Call `bm.edges.ensure_lookup_table()`, `bm.faces.ensure_lookup_table()`, `bm.verts.ensure_lookup_table()`
    - Count: non-manifold edges (`not e.is_manifold`), n-gons (`len(f.verts) > 4`), tris (`len(f.verts) == 3`), quads (`len(f.verts) == 4`), E-poles (`len(v.link_edges) == 3 and not v.is_boundary`), N-poles (`len(v.link_edges) >= 5 and not v.is_boundary`), loose verts (`len(v.link_edges) == 0`), loose edges (`len(e.link_faces) == 0`), boundary edges (`e.is_boundary`)
    - Calculate edge flow metric: average face angle at edges with 2 linked faces
    - Always call `bm.free()` (use try/finally)
    - Return dict with all metrics + grade + issues list
    - Use `_compute_grade(metrics)` helper with thresholds from research:
      - F: nm>20 or ngon%>25 or pole%>50 or loose>50
      - E: nm>5 or ngon%>10 or pole%>30 or loose>10
      - D: nm>0 or ngon%>5 or pole%>20 or loose>0 or tri%>40
      - C: ngon%>2 or pole%>10 or tri%>20
      - B: ngon%>0 or pole%>5 or tri%>10
      - A: everything else
    - Use `_list_issues(metrics)` helper to generate human-readable issue descriptions

    **2. handle_auto_repair(params: dict) -> dict** (MESH-02)
    - Accept `object_name`, optional `merge_distance` (default 0.0001), optional `max_hole_sides` (default 8)
    - Create bmesh from object
    - Chain repair operations in this exact order (order matters per research):
      1. Remove loose verts: `bmesh.ops.delete(bm, geom=loose_verts, context='VERTS')`
      2. Remove loose edges: `bmesh.ops.delete(bm, geom=loose_edges, context='EDGES')`
      3. Dissolve degenerate: `bmesh.ops.dissolve_degenerate(bm, dist=0.0001, edges=bm.edges[:])`
      4. Remove doubles: `bmesh.ops.remove_doubles(bm, verts=bm.verts[:], dist=merge_distance)`
      5. Recalculate normals: `bmesh.ops.recalc_face_normals(bm, faces=bm.faces[:])`
      6. Fill holes: `bmesh.ops.holes_fill(bm, edges=boundary_edges, sides=max_hole_sides)` (only if boundary edges exist)
    - Write back: `bm.to_mesh(obj.data); obj.data.update(); bm.free()`
    - Run `_quick_grade(obj)` after repair (reuses _compute_grade on fresh analysis)
    - Return report dict with counts for each operation + post_repair_grade

    **3. handle_check_game_ready(params: dict) -> dict** (MESH-08)
    - Accept `object_name`, optional `poly_budget` (default 50000), optional `platform` ("pc"/"mobile"/"console")
    - Run handle_analyze_topology internally for topology grade
    - Check topology grade (pass = A/B/C)
    - Calculate estimated triangle count: tris + quads*2 + ngons*3
    - Check poly budget
    - Check UV presence and overlaps (if UV layer exists, analyze via bmesh UV layer; if not, fail UV check)
    - Check materials (at least 1 material slot)
    - Check naming convention (not default Blender names like "Cube", "Sphere", "Cylinder")
    - Check transforms (applied = location near zero, rotation near zero, scale near 1.0)
    - Return structured dict: `{object_name, game_ready: bool, checks: {topology, poly_budget, uv, materials, naming, transform}, summary}`

    - Test the grading thresholds exhaustively: create metric dicts at each grade boundary and verify correct grade.
    - Test game_readiness_logic by mocking the topology analysis output and checking pass/fail conditions.
