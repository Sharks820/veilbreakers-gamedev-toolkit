    Extend `Tools/mcp-toolkit/blender_addon/handlers/mesh.py` with 5 new handler functions. Add them BELOW the existing handle_check_game_ready function (do not modify existing functions).

    **1. handle_select_geometry(params: dict) -> dict** (MESH-03)
    - Accept `object_name` and selection criteria: `material_index`, `material_name`, `vertex_group`, `face_normal_direction` (list[float]), `normal_threshold` (default 0.7), `loose_parts` (bool)
    - Create bmesh from object
    - Deselect all first: set `.select = False` on all verts, edges, faces
    - Apply selection criteria (can combine multiple):
      - **material_index/material_name**: Select faces where `f.material_index` matches. If `material_name` given, resolve to index via `obj.data.materials.find(name)`.
      - **vertex_group**: Get deform layer via `bm.verts.layers.deform.active`. Resolve group name to index via `obj.vertex_groups[name].index`. Select verts where `weights[group_index] > 0.0`, then select their linked faces.
      - **face_normal_direction**: Normalize direction vector, select faces where `f.normal.dot(direction) > threshold`.
      - **loose_parts**: Select verts with `len(v.link_faces) == 0`.
    - Write selection state back: `bm.to_mesh(obj.data); obj.data.update(); bm.free()`
    - Return: `{object_name, selected_faces, selected_verts, selected_edges, criteria_used}`

    **2. handle_edit_mesh(params: dict) -> dict** (MESH-06)
    - Accept `object_name`, `operation` (Literal: "extrude", "inset", "mirror", "separate", "join"), plus operation-specific params
    - **extrude**: Use `bmesh.ops.extrude_face_region(bm, geom=selected_faces)` then translate extruded verts by `offset` param (default [0,0,0.5]). Requires selected faces.
    - **inset**: Use `bmesh.ops.inset_region(bm, faces=selected_faces, thickness=thickness, depth=depth)` with params `thickness` (default 0.1), `depth` (default 0.0).
    - **mirror**: Use `bmesh.ops.mirror(bm, geom=bm.verts[:]+bm.edges[:]+bm.faces[:], axis=axis, mirror_u=True, mirror_v=True)` with `axis` param ("X"/"Y"/"Z", default "X"). Map axis string to matrix.
    - **separate**: Enter edit mode with context override, run `bpy.ops.mesh.separate(type=separate_type)` with `separate_type` param ("SELECTED"/"MATERIAL"/"LOOSE", default "SELECTED"), return to object mode.
    - **join**: Accept `object_names` (list of names to join into the target). Select all named objects, set target as active, run `bpy.ops.object.join()` with context override.
    - Write back for bmesh operations: `bm.to_mesh(obj.data); obj.data.update(); bm.free()`
    - Return: `{object_name, operation, vertex_count, face_count}` (post-operation counts)

    **3. handle_boolean_op(params: dict) -> dict** (MESH-05)
    - Accept `object_name` (target), `cutter_name`, `operation` ("UNION"/"DIFFERENCE"/"INTERSECT", default "DIFFERENCE"), `remove_cutter` (default True)
    - Validate both objects exist and are MESH type
    - Add BooleanModifier to target: `mod = target.modifiers.new(name="Boolean", type='BOOLEAN')`
    - Set `mod.operation = operation.upper()`, `mod.object = cutter`, `mod.solver = 'EXACT'`
    - Apply modifier via context override: `bpy.ops.object.modifier_apply(modifier=mod.name)`
    - If `remove_cutter` is True, `bpy.data.objects.remove(cutter, do_unlink=True)`
    - Return: `{object_name, operation, vertex_count, face_count}`

    **4. handle_retopologize(params: dict) -> dict** (MESH-07)
    - Accept `object_name`, `target_faces` (default 4000), `preserve_sharp` (default True), `preserve_boundary` (default True), `smooth_normals` (default True), `use_symmetry` (default False), `seed` (default 0)
    - Store before counts: `before_faces, before_verts`
    - Set object as active, select it, ensure object mode
    - Run `bpy.ops.object.quadriflow_remesh(...)` with context override and all params
    - Catch RuntimeError for "canceled" and suggest running auto_repair first
    - Return: `{object_name, before: {vertices, faces}, after: {vertices, faces}, target_faces, reduction_ratio, preserve_sharp}`

    **5. handle_sculpt(params: dict) -> dict** (MESH-04)
    - Accept `object_name`, `operation` ("smooth"/"inflate"/"flatten"/"crease"), `strength` (default 0.5), `iterations` (default 3)
    - **smooth**: Use bmesh (no mode switch needed). `bmesh.ops.smooth_vert(bm, verts=selected_or_all, factor=strength, ...)`. Write back.
    - **inflate/flatten/crease**: Require sculpt mode. Use context override to switch to SCULPT mode, run `bpy.ops.sculpt.mesh_filter(type=filter_type, strength=strength, iteration_count=iterations)`, switch back to OBJECT mode.
      - inflate -> filter_type="INFLATE"
      - flatten -> filter_type="SURFACE_SMOOTH" with `surface_smooth_shape_preservation=0.5`
      - crease -> filter_type="SHARPEN" with `sharpen_smooth_ratio=0.35`
    - Return: `{object_name, operation, strength, iterations}`

    - Test `_parse_selection_criteria` helper (if extracted) or test the criteria validation logic
    - Test that known operations are accepted and unknown ones raise ValueError
    - Test the axis mapping for mirror (X->0, Y->1, Z->2)
    - Do NOT test actual bmesh/bpy operations -- those require Blender. Mark with `@pytest.mark.blender`.
