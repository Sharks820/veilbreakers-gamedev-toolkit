**A. Extend _dungeon_gen.py** (WORLD-06): Add multi-floor dungeon generation:

1. **generate_multi_floor_dungeon** function: Params: width, height, num_floors (int=3), min_room_size, max_depth, cell_size, wall_height, connection_types (list of "staircase"/"elevator"/"ladder"/"pit_drop"), seed. Algorithm:
   - Generate staircase positions FIRST (shared connection_points list)
   - For each floor, call existing generate_dungeon_grid with constraint that walkable cells exist at staircase endpoints
   - Each floor offset by wall_height * floor_index on Y axis
   - Returns dict with "floors" (list of per-floor grids), "connections" (list of dicts: from_floor, to_floor, position, type), "total_rooms" count

2. **_place_connection_points** helper: Given grid dimensions, places 1-2 staircase positions per floor transition, ensuring they fall within room cells (not corridors or walls).

**B. Add pure-logic functions to worldbuilding_layout.py** for testable world design:

1. **generate_world_graph** (WORLD-04): Params: locations (list of dicts with name, type, position), target_distance (float=105.0), seed. Returns WorldGraph dataclass with nodes and edges. Validates ~105m walking distance between connected POIs. Uses simple proximity-based MST then adds extra edges for loop paths.

2. **generate_boss_arena_spec** (WORLD-03): Params: arena_type (str="circular"/"rectangular"), diameter (float=30.0), cover_count (int=4), hazard_zones (int=2), has_fog_gate (bool=True), phase_trigger_count (int=3). Returns dict with arena geometry spec, cover positions, hazard zone positions, fog gate position, phase trigger positions. Pure-logic, no bpy.

3. **generate_location_spec** (WORLD-01): Params: location_type (str), building_count (int), path_count (int), poi_count (int), seed. Returns dict with building positions, path waypoints, POI markers, terrain bounds. Composes building placement + path routing + POI distribution.

4. **generate_linked_interior_spec** (WORLD-05): Params: building_exterior_bounds, interior_rooms (list), door_positions. Returns dict with door_trigger markers, occlusion_zone bounds, lighting_transition fade zones. Pure-logic spec generation.
