```python
def handle_generate_hearthvale(params: dict) -> dict:
    """Generate Hearthvale fortified castle-town via generate_settlement().

    Uses the fully-featured settlement path (perimeter walls, interiors,
    lighting) rather than the older generate_town_layout() path.
    """
    from .settlement_generator import generate_settlement
    seed = params.get("seed", 3810)
    center = tuple(params.get("center", [0.0, 0.0]))
    radius = float(params.get("radius", 65.0))
    layout_brief = params.get(
        "layout_brief",
        "fortified castle-town, winding cobblestone, market square at center, "
        "radial streets from square, military quarter, commerce district",
    )
    result = generate_settlement(
        settlement_type="hearthvale",
        seed=seed,
        center=center,
        radius=radius,
        wall_height=3.5,       # Floor storey height (per generate_settlement signature)
        layout_brief=layout_brief,
    )
    result["hearthvale_wall_height_m"] = 5.5   # For Blender wall handler
    return result
```

### Task 3: Visual QA Pass -- Full Contact Sheet + Issue Resolution
type="auto"

**What:** Run the mandatory full visual QA pass: overhead + 4 ground-level angles on the complete town. Fix every detected issue: floating props, z-fighting at wall bases, missing faces, improper rotations, terrain mismatches. Run `blender_mesh game_check` on all 14 building objects. Hearthvale is the quality benchmark -- it must look amazing with zero visual defects before export.

**Implementation:**

1. Generate full contact sheet via `blender_viewport action=contact_sheet` with a minimum of 6 angles:
   - Overhead (top-down) -- full town overview
   - South approach -- main gate and fortress walls dominant
   - West approach -- exterior wall with guard towers
   - Market square level -- ground-level interior of market (stalls, fountain, cobblestones)
   - Tavern exterior + street -- warm lantern light, shop signs, cobblestones
   - Tavern interior (if accessible) or town hall interior -- AAA interior density verification

2. Examine contact sheet output. Fix each detected issue category:
   - **Floating objects**: Use `blender_execute` to snap object Z to terrain surface (`ob.location.z = max(terrain_z, 0)`)
   - **Z-fighting at wall bases**: Add a 0.05m vertical offset to wall segment base objects via `blender_execute`
   - **Missing perimeter segments**: Identify gap positions from overhead view, call `blender_execute` to instantiate additional wall segments
   - **Misaligned Tripo props**: Use `blender_execute` to rotate/reposition any stalls that face incorrect directions
   - **Material issues on Tripo imports**: Re-run `asset_pipeline action=cleanup_ai_model` with `load_extracted_textures=True` on any props showing blank textures

3. Run `blender_mesh action=game_check` on every building object (14 total). For each failure:
   - Non-manifold edges: `blender_mesh action=repair`
   - High poly count warning: note for LOD pass in Task 4 -- do NOT reduce poly count here (D-12: quality first)
   - Missing UVs: `blender_uv action=unwrap`
   - Inverted normals: `blender_mesh action=repair`

   - Wall height should read as imposing (5-6m relative to character scale)
   - Arrow slit windows should be visible
   - Battlements (crenellations) at top of walls
   - If walls appear as plain boxes without battlement detail, use `blender_execute` to add battlement geometry or call `blender_quality action=apply_style style=fortress` on wall objects

   - 2+ light sources per room

6. Run final contact sheet after all fixes. All 6 angles must show:
   - No floating objects
   - No z-fighting
   - No missing faces
   - Proportionally imposing fortress walls relative to buildings
   - Dense market square with stalls and fountain
   - Warm interior glow visible through windows

**Done criteria:**

- 6-angle contact sheet shows zero visual defects
- Fortress walls read as imposing 5-6m stone

### Task 4: Performance Profiling + Optimization Pass
type="auto"

**What:** Per D-12, generate at full quality first, then run the LOD pass and texture atlas optimization. Target: 60fps at 1080p, <5s load. Profile via `unity_performance action=profile_scene` after Unity import (or pre-export estimate via `blender_execute` poly count analysis). Generate LODs for all buildings and major props. Create a texture atlas for the cobblestone road tiles.

**Implementation:**

1. Pre-export polygon budget analysis via `blender_execute`:
   - Iterate all objects, sum tri count per district group
   - Log: market district total, civic district total, residential total, fortifications total
   - Flag any single object over 50,000 tris (LOD priority objects)

2. Generate LODs for all buildings and major props:
   - Call `asset_pipeline action=generate_lods` for each building object (14 total)
   - Call `asset_pipeline action=generate_lods` for each Tripo stall (5+ stalls)
   - Call `asset_pipeline action=generate_lods` for fountain
   - LOD distances: LOD0 = <20m, LOD1 = 20-60m, LOD2 = 60-120m, LOD3 = billboard/imposter > 120m
   - Target: LOD1 at 50% tri reduction, LOD2 at 85% tri reduction

3. Texture atlas optimization for road cobblestones:
   - The cobblestone PBR material tiles across all road segments. Ensure UV scale is correct (1 tile = 1m real-world) to avoid stretching visible at ground level
   - If multiple road segment objects use identical materials, merge to single atlas via `blender_uv action=pack_atlas`

4. Verify Tripo prop poly counts are within budget:
   - Target: each market stall < 8,000 tris at LOD0
   - Fountain < 12,000 tris at LOD0
   - If over budget, reduce via `blender_mesh action=decimate ratio=0.7` on LOD1+ variants only -- preserve LOD0

5. Vegetation around exterior (Claude's discretion):
   - Add orchard trees along southern exterior (river bank side), scattered farmland plots on outskirts lots
   - Use `blender_worldbuilding action=generate_vegetation` if available, or `blender_execute` with tree/grass scatter
   - Vegetation is LOD-critical -- use billboard LOD2+ for trees visible at distance

6. Export scene stats summary: total tri count per district, total unique materials, total texture memory estimate (sum texture resolutions). Document in a brief `hearthvale_stats.txt` in the export directory.

**Done criteria:**
- LODs generated for all 14 buildings and 5+ stalls
- No single LOD0 object over 50,000 tris (flag if found, do not silently ignore)
- Cobblestone road UVs tile correctly at 1m scale
- Scene stats documented: total tri count, material count, texture memory estimate
- LOD distances set (20m / 60m / 120m thresholds)
