**7. generate_tutorial_system_script(name, steps=None, namespace="") -> tuple[str, str, str, str]**
Returns (data_so_cs, manager_cs, uxml, uss).
- data_so_cs: ScriptableObject with [CreateAssetMenu(menuName = "VeilBreakers/Tutorial/Step")] containing:
  - string stepTitle, string stepDescription, string highlightObjectPath, Rect highlightRect
  - string requiredAction (e.g., "press_attack", "open_inventory"), bool isOptional
- manager_cs (MonoBehaviour): Step-based state machine:
  - Current step index, advance on action trigger callback
  - Shows tooltip overlay at highlight rect position
  - Dims background except highlighted area (semi-transparent overlay panel)
  - PrimeTween: Tween.Alpha for fade transitions between steps
  - Public methods: StartTutorial(), AdvanceStep(), SkipTutorial()
  - Events: OnTutorialComplete, OnStepChanged
- uxml: Tutorial overlay layout (tooltip, highlight frame, skip button, step counter)
- uss: Dark fantasy themed styling (Cinzel font, dark overlays, gold accents)

**8. generate_accessibility_script(name, namespace="") -> tuple[str, str, str]**
Returns (settings_cs, shader_hlsl, renderer_feature_cs).
- settings_cs (MonoBehaviour + PlayerPrefs persistence):
  - Colorblind mode enum: None, Protanopia, Deuteranopia, Tritanopia
  - Subtitle text scale: float (1.0 to 3.0, default 1.0)
  - Screen reader tags: bool (enable/disable ARIA-like labels on UI elements)
  - Motor accessibility: toggle vs hold option, input timing multiplier (0.5-2.0)
  - Auto-save settings to PlayerPrefs
  - Public property for each setting with change event
- shader_hlsl: Fullscreen colorblind simulation shader
  - 3x3 color transformation matrices for Protanopia, Deuteranopia, Tritanopia (use exact values from RESEARCH.md)
  - Integer uniform _ColorblindMode (0=off, 1=protanopia, 2=deuteranopia, 3=tritanopia)
  - Convert sRGB to linear before matrix multiply, convert back after
  - URP-compatible fullscreen blit shader (Shader "VeilBreakers/Accessibility/ColorblindFilter")
- renderer_feature_cs: URP ScriptableRendererFeature
  - Uses RecordRenderGraph API (not legacy Execute) per Phase 10 SHDR-02 pattern
  - Reads _ColorblindMode from material property
  - Fullscreen pass applying the colorblind shader

**9. generate_character_select_script(hero_paths=None, namespace="") -> tuple[str, str, str, str]**
Returns (data_so_cs, manager_cs, uxml, uss).
- Default hero_paths: ["IRON", "VENOM", "SURGE", "DREAD", "BLAZE"]
- data_so_cs: ScriptableObject with hero path data:
  - string pathName, string description, Sprite icon, Color themeColor
  - string[] startingAbilities, int baseStrength, baseAgility, baseIntelligence
- manager_cs (MonoBehaviour):
  - Carousel navigation through hero paths (left/right buttons)
  - Appearance customization: skin color, hair style, armor tint (string arrays for options)
  - Name entry: InputField with validation (3-20 chars, alphanumeric + spaces)
  - "Embark" button to confirm and start game
  - PrimeTween: Sequence.Create() for carousel transition animations
  - Events: OnPathSelected, OnCharacterCreated
- uxml: Character select layout (path carousel, appearance panel, name input, embark button)
- uss: Dark fantasy styling matching VeilBreakers theme

**10. generate_world_map_script(name, map_resolution=512, fog_resolution=256, namespace="") -> tuple[str, str]**
Returns (editor_cs, runtime_cs).
- editor_cs: Generates 2D map texture from TerrainData:
  - TerrainData.GetHeights() to sample terrain heights
  - Color mapping: low=dark green, mid=brown, high=gray/white (snow)
  - Water level threshold for blue areas
  - Save as PNG texture asset
  - MenuItem: [MenuItem("VeilBreakers/World/Generate World Map")]
- runtime_cs (MonoBehaviour):
  - Displays generated map texture on UI RawImage
  - Fog-of-war: separate Texture2D mask (fog_resolution), initially all black
  - Reveal circle around player as they explore (configurable reveal_radius)
  - Location markers: List of (world position, icon, label) rendered as UI overlays
  - Player position blip: converts 3D world position to 2D map UV coordinates
  - Zoom and pan controls
  - Smooth fog reveal via PrimeTween alpha transition

**11. generate_rarity_vfx_script(name, namespace="") -> str**
Returns single MonoBehaviour C# string.
- RARITY_VFX dict with 5 tiers: Common (gray, no glow, no particles), Uncommon (green, 0.3 glow, 5 particles), Rare (blue, 0.6 glow, 15 particles), Epic (purple, 0.8 glow, 30 particles), Legendary (gold, 1.2 glow, 60 particles)
- MonoBehaviour that:
  - Takes ItemRarity enum (0-4) as input
  - Configures attached ParticleSystem emission rate per tier
  - Sets material emission color + intensity via MaterialPropertyBlock
  - Legendary tier: additional particle system with unique gold sparkle effect
  - Public method: SetRarity(int rarityTier)
  - Uses enum: Common=0, Uncommon=1, Rare=2, Epic=3, Legendary=4

**12. generate_corruption_vfx_script(name, namespace="") -> str**
Returns single MonoBehaviour C# string.
- MonoBehaviour that visualizes 0-100% corruption on equipment:
  - Material property _CorruptionAmount (0.0 to 1.0) drives:
    - Vein pattern intensity: material.SetFloat("_VeinIntensity", corruption * 1.5f)
    - Color shift: Lerp from original color to dark purple (0.3, 0.1, 0.4) based on corruption
    - Emission: material.SetColor("_EmissionColor", veinColor * corruption * 2f)
  - ParticleSystem emission rate scales with corruption (0 at 0%, maxRate at 100%)
  - Threshold effects:
    - 25%: subtle vein patterns appear
    - 50%: color shift becomes noticeable, occasional particle bursts
    - 75%: strong vein glow, constant particle emission
    - 100%: maximum distortion, pulsing glow effect via PrimeTween
  - Public method: SetCorruption(float amount) -- clamps 0-1
  - 100% pulse: Tween.Custom(0f, 1f, 1.0f, Ease.InOutSine, cycles: -1, cycleMode: CycleMode.Yoyo) for breathing glow
