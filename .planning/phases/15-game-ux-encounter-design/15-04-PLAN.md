Create async handler functions for each action following the established _handle_camera_* pattern:
- _handle_ux_minimap: calls generate_minimap_script, writes tuple[0] to Assets/Editor/Generated/UX/, tuple[1] to Assets/Scripts/Runtime/UX/
- _handle_ux_damage_numbers: calls generate_damage_numbers_script, writes to Assets/Scripts/Runtime/UX/
- _handle_ux_interaction_prompts: writes to Assets/Scripts/Runtime/UX/
- _handle_ux_primetween_sequence: writes to Assets/Scripts/Runtime/UI/
- _handle_ux_tmp_font_asset: writes to Assets/Editor/Generated/UX/
- _handle_ux_tmp_components: writes to Assets/Editor/Generated/UX/
- _handle_ux_tutorial: calls generate_tutorial_system_script (4-tuple), writes SO to Assets/ScriptableObjects/UX/, manager to Assets/Scripts/Runtime/UX/, UXML to Assets/UI/Tutorial/, USS to Assets/UI/Tutorial/
- _handle_ux_accessibility: calls generate_accessibility_script (3-tuple), writes settings to Assets/Scripts/Runtime/UX/, shader to Assets/Shaders/, renderer feature to Assets/Scripts/Runtime/Rendering/
- _handle_ux_character_select: 4-tuple dispatch similar to tutorial
- _handle_ux_world_map: 2-tuple dispatch to Editor/ and Runtime/
- _handle_ux_rarity_vfx: single file to Assets/Scripts/Runtime/UX/
- _handle_ux_corruption_vfx: single file to Assets/Scripts/Runtime/UX/

Each handler returns JSON with: files_written, next_steps (always includes "recompile" step).
