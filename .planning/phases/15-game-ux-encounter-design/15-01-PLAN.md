Create Tools/mcp-toolkit/src/veilbreakers_mcp/shared/unity_templates/ux_templates.py with these 6 generators following the established lines-based string concatenation pattern:

**1. generate_minimap_script(name, render_texture_size=256, zoom=50.0, follow_target="Player", culling_layers=None, compass_enabled=True, marker_types=None, update_interval=3, namespace="") -> tuple[str, str]**
Returns (editor_cs, runtime_cs).
- editor_cs: Creates orthographic Camera GameObject positioned above scene, creates RenderTexture asset, adds RawImage to Canvas
- runtime_cs (MonoBehaviour):
  - CRITICAL: Uses Camera.orthographic = true with orthographicSize = zoom
  - CRITICAL: Follows player transform position EXACTLY (cam.transform.position = new Vector3(target.x, cam.transform.position.y, target.z)) for 1:1 positional accuracy
  - Camera renders to RenderTexture, displayed via RawImage
  - Culling mask set to specified layers (default: "Minimap", "Terrain")
  - Camera clearFlags = CameraClearFlags.SolidColor with dark background
  - Camera height 100f above player
  - Camera rotation Quaternion.Euler(90, 0, 0) (top-down view)
  - World-space markers: Dictionary<string, Transform> tracking POIs/NPCs/objectives
  - Marker icons rendered on minimap via UI Image overlays positioned by world-to-viewport conversion
  - Optional compass ring with rotation matching player facing direction
  - Update frequency: every N frames (default 3) for performance
  - configurable zoom (orthographicSize) via public property

**2. generate_damage_numbers_script(name, pool_size=20, float_height=80.0, duration=0.8, crit_scale=1.5, namespace="") -> str**
Returns single MonoBehaviour C# string.
- Uses ObjectPool<GameObject> (UnityEngine.Pool) -- pre-allocate pool_size damage number prefabs
- Each damage number: TextMeshPro (world space or screen space overlay) showing damage value
- Color coding: Dictionary<string, Color> mapping damage type names to colors (use VeilBreakers 10 brand colors: IRON=gray, VENOM=green, SURGE=blue, DREAD=purple, BLAZE=orange, FROST=cyan, VOID=dark purple, HOLY=gold, NATURE=forest green, SHADOW=dark gray)
- Crit hits: scale text by crit_scale, add "!" suffix
- Animation via PrimeTween (NEVER DOTween):
  - Sequence.Create()
    .Group(Tween.UIAnchoredPosition(rect, endValue: start + Vector2.up * float_height, duration, Ease.OutCubic))
    .Group(Tween.Alpha(canvasGroup, 0f, duration, Ease.InQuad))
    .OnComplete(() => ReturnToPool(obj))
- Random X offset (-20 to 20) to prevent stacking
- Public method: ShowDamage(Vector3 worldPos, float amount, string damageType, bool isCrit)

**3. generate_interaction_prompts_script(name, prompt_text="Interact", trigger_radius=2.5, fade_duration=0.3, namespace="") -> str**
Returns single MonoBehaviour C# string.
- Sphere trigger collider (radius = trigger_radius) on interactable objects
- OnTriggerEnter/Exit to show/hide prompt UI
- Prompt UI: Canvas with TextMeshProUGUI showing "[Key] Action"
- Key display: Uses InputAction reference + GetBindingDisplayString() for dynamic rebind support
- PrimeTween fade animation: Tween.Alpha(canvasGroup, 1f, fade_duration) on show, Tween.Alpha(canvasGroup, 0f, fade_duration) on hide
- Billboard effect: prompt always faces camera
- Configurable action text per instance via SerializeField

**4. generate_primetween_sequence_script(sequence_type="panel_entrance", name="default", namespace="") -> str**
Returns C# utility class with static methods for common UI animation patterns.
- sequence_type selects preset: "panel_entrance", "panel_exit", "button_hover", "notification_popup", "screen_shake", "damage_flash", "item_pickup", "level_up"
- All methods use PrimeTween API exclusively:
  - using PrimeTween;
  - Tween.Scale, Tween.Alpha, Tween.UIAnchoredPosition
  - Sequence.Create().Chain().Group()
  - Tween.PunchLocalScale, Tween.ShakeLocalPosition
  - Ease.OutBack, Ease.OutCubic, Ease.InQuad etc.
- NEVER generate DOTween code (no DOTween.To, no transform.DOScale, no using DG.Tweening)
- Returns full class with multiple static methods, each returning a Sequence or Tween

**5. generate_tmp_font_asset_script(font_path="Assets/Fonts/Cinzel-Regular.ttf", output_path="Assets/Fonts/Generated", sampling_size=48, atlas_width=1024, atlas_height=1024, character_set=None, namespace="") -> str**
Returns editor C# script.
- Uses TMP_FontAsset.CreateFontAsset(sourceFont, sampling_size, 5, GlyphRenderMode.SDFAA, atlas_width, atlas_height)
- Default character_set: ASCII 32-126 + extended Latin (accented chars)
- TryAddCharacters for the character set
- AssetDatabase.CreateAsset to save font asset
- MenuItem for easy access: [MenuItem("VeilBreakers/Fonts/Generate Font Asset")]
- Configurable fallback font chain via serialized list

**6. generate_tmp_component_script(name, font_asset_path="", font_size=36, color=None, rich_text=True, auto_sizing=False, min_size=18, max_size=72, namespace="") -> str**
Returns editor C# script that configures TMP components.
- Creates or configures TextMeshProUGUI components on selected GameObjects
- Sets font asset, size, color, rich text enabled, auto sizing with min/max
- Font fallback chain setup
- Supports alignment, overflow mode, word wrapping options
- MenuItem: [MenuItem("VeilBreakers/UI/Setup TMP Components")]

All generators MUST:
- Use line-based concatenation (lines = [], lines.append, "\n".join(lines))
- Include _sanitize_cs_string for any user-provided string parameters
- Support optional namespace parameter with indent wrapping
- Include proper using statements (UnityEngine, TMPro, PrimeTween, UnityEngine.Pool, UnityEngine.InputSystem where needed)
