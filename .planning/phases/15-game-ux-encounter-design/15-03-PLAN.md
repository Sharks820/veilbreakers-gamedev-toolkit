**1. generate_encounter_system_script(name, namespace="") -> tuple[str, str]**
Returns (wave_data_so_cs, encounter_manager_cs).
- wave_data_so_cs: ScriptableObject [CreateAssetMenu(menuName = "VeilBreakers/Encounter/Wave Definition")]
  - [Serializable] class SpawnEntry: string prefabName, int count, float spawnDelay, float difficultyMultiplier
  - SpawnEntry[] entries, float waveDelay, string[] victoryConditions (all_dead, timer, objective, boss_dead)
  - float timeLimit (optional, for timer condition)
  - string defeatCondition (player_death, time_expired, objective_failed)
- encounter_manager_cs (MonoBehaviour):
  - SerializeField List<VB_WaveDefinition> _waves
  - Trigger volume entry point (OnTriggerEnter with "Player" tag check)
  - Coroutine-based wave spawning: StartCoroutine(RunEncounter())
  - Track spawned enemies count, defeated count
  - Victory check per wave: "all_dead" = defeated >= spawned, "timer" = elapsed >= timeLimit
  - Defeat check: player HP or time expired
  - Events: UnityEvent OnEncounterStarted, OnWaveStarted(int waveIndex), OnWaveComplete, OnEncounterVictory, OnEncounterDefeat
  - AI director hook: public void SetDifficultyModifier(float modifier) adjusts spawn entry difficultyMultiplier
  - Public methods: StartEncounter(), ResetEncounter(), ForceEndEncounter()
  - Fog gate support: SerializeField GameObject[] _fogGates (enable on start, disable on victory)

**2. generate_ai_director_script(name, namespace="") -> str**
Returns single MonoBehaviour C# string.
- Performance tracking:
  - int _recentDeaths (last N encounters), float _averageClearTime, float _damageTakenRatio
  - Sliding window: List<float> _clearTimes (last 5 encounters) for moving average
  - Track per-encounter: kills, deaths, damage dealt, damage taken, time elapsed
- Difficulty calculation:
  - float _difficultyScore (0.0 to 1.0, starting at 0.5)
  - SerializeField AnimationCurve _spawnRateCurve (maps difficultyScore to spawn rate multiplier)
  - SerializeField AnimationCurve _enemyStatCurve (maps difficultyScore to enemy stat multiplier)
  - float _adjustmentSpeed = 0.1f (lerp speed toward target difficulty)
  - Target calculation: decrease on deaths, increase on fast clears and low damage taken
- Public API:
  - float GetSpawnRateMultiplier() => _spawnRateCurve.Evaluate(_difficultyScore)
  - float GetStatMultiplier() => _enemyStatCurve.Evaluate(_difficultyScore)
  - void RecordEncounterResult(float clearTime, int deaths, float damageRatio)
  - void UpdateDifficulty() -- called in Update, lerps toward target
  - float DifficultyScore { get; } -- read-only property
- Integration: finds EncounterManager instances and calls SetDifficultyModifier()

**3. generate_encounter_simulator_script(name, namespace="") -> str**
Returns editor C# string (EditorWindow, IMGUI).
- MUST have: #if UNITY_EDITOR / using UnityEditor;
- Window: [MenuItem("VeilBreakers/Tools/Encounter Simulator")]
- Parameters (IMGUI fields):
  - int simulationCount = 1000 (number of Monte Carlo runs)
  - float playerDPS, playerHP, playerDodgeChance
  - WaveDefinition reference (SO field) or manual wave config
  - AI director difficulty score slider (0-1)
- Simulation loop (NOT on main thread -- use EditorApplication.update with per-frame batching):
  - For each run: simulate wave spawning, apply player DPS vs enemy HP, track kills/deaths/time
  - Simple combat model: player deals DPS * deltaTime damage, enemies deal their DPS * deltaTime * (1-dodgeChance)
  - Run ends on player death or all enemies defeated
- Results display (IMGUI):
  - Win rate: wins / total runs as percentage
  - Average clear time: mean + standard deviation
  - Average player HP remaining: mean
  - Average DPS dealt: total damage / time
  - Difficulty recommendation: "Too Easy" / "Balanced" / "Too Hard" based on win rate thresholds
- Export button: writes results to JSON file

**4. generate_boss_ai_script(name, phase_count=3, namespace="") -> str**
Returns single MonoBehaviour C# string.
- Phase system:
  - enum BossPhase { Phase1, Phase2, Phase3, ..., Enrage } (generated based on phase_count)
  - SerializeField float _phase2Threshold = 0.7f (70% HP)
  - SerializeField float _phase3Threshold = 0.35f (35% HP)
  - SerializeField float _enrageTimer = 180f (3 minutes)
  - BossPhase _currentPhase, BossPhase _targetPhase, bool _pendingTransition
- Phase transition:
  - CheckPhaseTransition() method: compare HP ratio against thresholds + enrage timer
  - Queue transition via _pendingTransition flag (don't interrupt current attack animation)
  - Execute transition on next state machine cycle: play transition animation, adjust stats
  - Per-phase stat multipliers: SerializeField float[] _phaseSpeedMultiplier, _phaseDamageMultiplier
- Attack patterns:
  - Per-phase attack set: SerializeField string[][] or List<AttackPattern>[] where AttackPattern has name, damage, range, cooldown, animTrigger
  - Select random attack from current phase's set
  - Attack cooldown tracking per attack
  - Enrage phase: reduced cooldowns, increased damage, faster movement
- State machine:
  - enum BossState { Idle, Attacking, PhaseTransition, Stunned, Dead }
  - FSM Update: check phase transitions, select and execute attacks, handle stun
- Integration points:
  - Delegates to DamageCalculator for damage (VB delegation pattern)
  - Events: UnityEvent<BossPhase> OnPhaseChanged, UnityEvent OnBossDeath, UnityEvent OnEnrage
  - Fog gate reference for arena lockdown
  - Health bar UI reference for phase indicator display

All generators MUST:
- Use line-based concatenation (lines = [], lines.append, "\n".join(lines))
- Include _sanitize_cs_string for user-provided name parameters
- Support optional namespace parameter
- Include proper using statements
