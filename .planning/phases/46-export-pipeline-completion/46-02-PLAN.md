         Gets mesh data from bpy.data.objects[object_name], runs bmesh.ops.convex_hull, applies decimate modifier if face count > max_faces, creates new object named "{object_name}_COL", links to same collection. Returns {"collision_object": name, "face_count": N, "original_object": object_name}.

    3. Create `blender_addon/handlers/vegetation_serializer.py`:
       - PURE-LOGIC function `serialize_vegetation_instances(vegetation_objects: list[dict], terrain_bounds: dict) -> dict`:
         - vegetation_objects: list of dicts with keys: name (str), location (tuple x,y,z in Blender Z-up), rotation_z (float, radians), scale (tuple x,y,z), veg_type (str, e.g. "Oak_Tree")
         - terrain_bounds: dict with keys: min_x, max_x, min_y, max_y (Blender XY plane)
         - Returns dict with:
           - "terrain_name": from terrain_bounds.get("terrain_name", "Terrain")
           - "tree_prototypes": list of {"prefab_name": veg_type, "prototype_index": N} for each unique veg_type
           - "instances": list of {"position": [nx, 0.0, ny], "rotation": rot_z, "width_scale": scale_x, "height_scale": scale_z, "prototype_index": idx}
         - COORDINATE SWAP: Blender Z-up -> Unity Y-up:
           - Blender (X, Y, Z) -> Unity: X stays, Blender Z -> Unity Y, Blender Y -> Unity Z
           - BUT positions are normalized to [0,1] on terrain XZ plane, so:
             nx = (blender_x - min_x) / (max_x - min_x)
             ny = (blender_y - min_y) / (max_y - min_y)
             Position in Unity: [nx, 0.0, ny] (Y=0 because height comes from terrain)
         - Rotation: use rotation_z directly (around vertical axis, same concept in both engines)
         - Scale: width_scale = average of scale_x and scale_y; height_scale = scale_z
       - BPY-DEPENDENT function `handle_serialize_vegetation(params: dict) -> dict`:
         Reads objects from Blender's "Vegetation" collection, extracts location/rotation/scale, calls serialize_vegetation_instances. Writes JSON to output_path. Returns {"output_path": ..., "instance_count": N, "prototype_count": M}.
