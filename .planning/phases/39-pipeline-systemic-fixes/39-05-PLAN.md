<objective>
Fix all square-terrain assumptions across 7+ files and rectangular terrain bugs (heightmap squash, road warp, scatter drift) so non-square terrains work correctly throughout the entire pipeline.

Purpose: Many terrain operations assume square grids using sqrt(vert_count) or max(dims.x, dims.y). This silently breaks non-square terrains causing chunking truncation, scatter drift, road warping, and heightmap squashing. Since real game terrains are rarely perfectly square, these bugs affect every downstream consumer.
Output: All terrain operations handle rectangular grids correctly. Tests verify non-square terrain behavior.
</objective>
