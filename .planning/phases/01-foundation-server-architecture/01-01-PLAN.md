    1. Create `Tools/mcp-toolkit/src/veilbreakers_mcp/shared/blender_client.py`:

       Class `BlenderConnection`:
       - `__init__(self, host: str = "localhost", port: int = 9876, timeout: int = 300)`
         stores host, port, timeout. Sets `_socket: socket.socket | None = None`
       - `def connect(self) -> None`:
         Creates `socket.AF_INET, socket.SOCK_STREAM`, calls `settimeout(self.timeout)`,
         `connect((self.host, self.port))`. On `ConnectionRefusedError`:
         raises `ConnectionError("Cannot connect to Blender on {host}:{port}. Start Blender and enable the VeilBreakers addon.")`
       - `def disconnect(self) -> None`:
         If `_socket` not None, `close()` and set to None
       - `def is_alive(self) -> bool`:
         Try sending ping command `{"type": "ping", "params": {}}`, return True if response
         status is "success". On any exception, return False.
       - `def reconnect(self) -> None`:
         Call `disconnect()` then `connect()`
       - `async def send_command(self, command_type: str, params: dict | None = None) -> dict`:
         Build `BlenderCommand(type=command_type, params=params or {})`.
         Serialize to JSON bytes. Send length-prefixed message:
         4 bytes big-endian length + JSON payload.
         Receive response with same length-prefix protocol.
         Parse as `BlenderResponse`. If status == "error", raise `BlenderCommandError(response)`.
         Return `response.result`.
       - `async def capture_viewport_bytes(self) -> bytes`:
         `send_command("get_viewport_screenshot", {"format": "png"})`.
         Response result contains `{"filepath": "/tmp/..."}`. Read file bytes and return.
       - `async def capture_viewport_to_file(self) -> str`:
         Same as above but return the filepath string.

       Class `BlenderCommandError(Exception)`:
       - Stores the `BlenderResponse` object
       - `__str__` returns `BlenderError.to_tool_response()` format

       IMPORTANT: The `send_command` method must use `asyncio.get_event_loop().run_in_executor(None, self._sync_send, ...)` internally because the actual socket operations are synchronous. Create a private `_sync_send(self, command_type: str, params: dict) -> dict` that does the actual socket I/O.

       Length-prefix protocol for the socket:
       - Send: `struct.pack(">I", len(json_bytes)) + json_bytes`
       - Receive: read 4 bytes for length via `struct.unpack(">I", ...)`, then read exactly that many bytes

    2. Create `Tools/mcp-toolkit/src/veilbreakers_mcp/blender_server.py`:

       ```python
       from mcp.server.fastmcp import FastMCP, Image, Context
       from veilbreakers_mcp.shared.blender_client import BlenderConnection
       from veilbreakers_mcp.shared.config import Settings

       settings = Settings()
       mcp = FastMCP(
           "veilbreakers-blender",
           description="VeilBreakers Blender game development tools",
       )

       _connection: BlenderConnection | None = None

       def get_blender_connection() -> BlenderConnection:
           global _connection
           if _connection is None or not _connection.is_alive():
               _connection = BlenderConnection(
                   host=settings.blender_host,
                   port=settings.blender_port,
                   timeout=settings.blender_timeout,
               )
               _connection.connect()
           return _connection

       # Placeholder tool (compound tools added in Plan 03)
       @mcp.tool()
       async def blender_ping() -> str:
           """Check if Blender is connected and responsive."""
           blender = get_blender_connection()
           result = await blender.send_command("ping")
           return f"Blender connection OK: {result}"

       def main():
           mcp.run(transport="stdio")

       if __name__ == "__main__":
           main()
       ```
