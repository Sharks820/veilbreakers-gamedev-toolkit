## Tasks

### Task 1: Wire Building Quality Generators into Grammar Details
type="auto"

Replace the 0.5m cube detail generation (lines 737-749 of _building_grammar.py) with proper geometry from building_quality.py generators:
- `timber_frame` -> generate_timber_frame() walls for medieval
- `chimney` -> generate_chimney() with brick/stone pattern
- `flying_buttress` -> proper buttress geometry (tapered pillar + angled strut)
- `gargoyle` -> sculpted creature perched on ledge
- `rose_window` -> generate_gothic_window(style="rose_window")
- `spire` -> pointed octagonal cone
- `battlement` / `machicolation` / `murder_hole` -> generate_battlements()
- `window_boxes` -> planter box with vegetation cluster
- `vine_growth` / `moss_patches` / `root_buttress` -> organic detail geometry
- `woodpile` -> stacked log geometry

Add new operation type "mesh_spec" to BuildingSpec that carries full vertex/face data from building_quality generators. Update _building_ops_to_mesh_spec() in worldbuilding.py to handle this type.

**Done when:** Each detail type produces distinct, recognizable geometry (not identical boxes). Gargoyle != buttress != chimney visually.

### Task 2: CGA-Style Facade Split Grammar
type="auto"

Implement recursive facade splitting in evaluate_building_grammar():
1. For each wall face: split(y, floors) -> split(x, bays) -> fill(bay_type)
2. Bay types: window, door, wall_panel, balcony (with style-dependent probabilities)
3. Floor-dependent rules: ground floor has doors + larger windows, upper floors have smaller windows
4. Per-building randomized bay count (within style's bay_divisor range +/- 1)
5. Corner bays always solid (structural)
6. Aligned windows across floors (column consistency)

The split grammar should produce window/door openings with positions that naturally vary between buildings (different bay counts, different fill patterns) while maintaining architectural coherence (aligned columns, consistent proportions).

**Done when:** Two buildings with same style + different seeds have visibly different facade layouts (different bay counts, different window placements) while both looking architecturally correct.

### Task 3: Proper Roof Generation
type="auto"

Replace the flat-box roof in evaluate_building_grammar() with calls to building_quality's generate_roof():
1. Map grammar roof types to building_quality roof styles: gabled->gable, pointed->hip, flat->flat, domed->conical_tower
2. Map grammar roof materials to building_quality materials: thatch->thatch, slate->slate, stone_parapet->tile
3. Add straight skeleton algorithm for hip roofs from rectangular footprints (simplified polyskel: offset inward at 45deg to find ridge line)
4. Roof operations now produce actual tile/shingle geometry via generate_roof()

Add new roof type "mansard" to STYLE_CONFIGS for a 6th roof variety.

**Done when:** Roofs have visible individual tile/shingle geometry. Gable roofs have triangular end walls. Hip roofs have 4 sloping faces meeting at a ridge.

### Task 4: Facade Variation System
type="auto"

Add seed-based variation to ensure no two buildings look identical:
1. Randomize floor height within +/- 15% of style default
2. Randomize bay count per wall: style's bay_divisor +/- 1 (minimum 2)
3. Randomize window size within +/- 20% of style default
4. Select random subset of style details (not all details on every building)
5. Randomize door position: center +/- 1 bay offset
6. Add color/material tint variation per building (slight hue/value shift in material name)
7. Track variation in BuildingSpec metadata for debugging

**Done when:** 5 buildings generated with seeds 1-5 in same style all have visibly different facades. Verified by checking that operations lists differ in length and content.
