Create these 14 generator functions:

1. `generate_asset_move_script(old_path: str, new_path: str) -> str`
   - CRITICAL: Uses AssetDatabase.MoveAsset, NEVER File.Move (File.Move breaks GUID tracking)
   - Creates destination directory if needed via AssetDatabase.CreateFolder
   - Reports old_path, new_path, guid in result JSON
   - MenuItem: `VeilBreakers/Assets/Move Asset`

2. `generate_asset_rename_script(asset_path: str, new_name: str) -> str`
   - Uses AssetDatabase.RenameAsset (preserves .meta/GUID)
   - MenuItem: `VeilBreakers/Assets/Rename Asset`

3. `generate_asset_delete_script(asset_path: str, safe_delete: bool = True) -> str`
   - If safe_delete=True: first scans for references using AssetDatabase.GetDependencies or GUID search. If references found, writes them to result JSON as warnings and does NOT delete. Returns validation_status="blocked_by_references"
   - If safe_delete=False: deletes directly via AssetDatabase.DeleteAsset
   - MenuItem: `VeilBreakers/Assets/Delete Asset`

4. `generate_asset_duplicate_script(source_path: str, dest_path: str) -> str`
   - Uses AssetDatabase.CopyAsset
   - MenuItem: `VeilBreakers/Assets/Duplicate Asset`

5. `generate_create_folder_script(folder_path: str) -> str`
   - Splits path and creates each level via AssetDatabase.CreateFolder (parent, child)
   - Example: "Assets/Prefabs/Monsters" -> CreateFolder("Assets/Prefabs", "Monsters")
   - MenuItem: `VeilBreakers/Assets/Create Folder`

6. `generate_fbx_import_script(asset_path: str, scale: float = 1.0, mesh_compression: str = "Off", animation_type: str = "None", import_animation: bool = False, normals_mode: str = "Import", import_blend_shapes: bool = True, optimize: bool = True, is_readable: bool = False, preset_type: str = "") -> str`
   - If preset_type is provided (hero/monster/weapon/prop/environment), use these defaults:
     - hero: scale=1.0, mesh_compression=Off, animation_type=Humanoid, import_animation=True, optimize=True
     - monster: scale=1.0, mesh_compression=Off, animation_type=Humanoid, import_animation=True, optimize=True
     - weapon: scale=1.0, mesh_compression=Medium, animation_type=None, import_animation=False, optimize=True
     - prop: scale=1.0, mesh_compression=Medium, animation_type=None, import_animation=False, optimize=True
     - environment: scale=1.0, mesh_compression=Low, animation_type=None, import_animation=False, optimize=True
   - C# casts AssetImporter.GetAtPath(assetPath) as ModelImporter
   - Sets: globalScale, meshCompression (ModelImporterMeshCompression enum), animationType (ModelImporterAnimationType enum), importAnimation, importNormals (ModelImporterNormals enum), importBlendShapes, isReadable, optimizeMeshPolygons, optimizeMeshVertices
   - Calls SaveAndReimport()
   - Does NOT conflict with existing BlenderFBXImportPostprocessor (this is explicit per-asset config, not an AssetPostprocessor)
   - MenuItem: `VeilBreakers/Assets/Configure FBX Import`

7. `generate_texture_import_script(asset_path: str, max_size: int = 2048, srgb: bool = True, mipmap: bool = True, filter_mode: str = "Bilinear", wrap_mode: str = "Repeat", sprite_mode: str = "", platform_overrides: dict[str, dict] | None = None, preset_type: str = "", auto_detect_srgb: bool = False) -> str`
   - If preset_type provided: hero=2048, monster=1024, weapon=1024, prop=512, ui=1024
   - If auto_detect_srgb=True: C# checks if asset name contains "normal", "roughness", "metallic", "ao", "height" (case-insensitive) and sets sRGB=false for those
   - Platform overrides: for each platform key (Standalone, Android, iOS, WebGL), calls GetPlatformTextureSettings/SetPlatformTextureSettings with overridden=true, format (TextureImporterFormat enum), max_size
   - Default platform compression if preset_type provided: Standalone=DXT5, Android=ASTC_6x6, iOS=ASTC_6x6
   - sprite_mode: if non-empty, sets textureType to Sprite, spriteImportMode
   - Calls SaveAndReimport()
   - MenuItem: `VeilBreakers/Assets/Configure Texture Import`

8. `generate_material_remap_script(fbx_path: str, remappings: dict[str, str]) -> str`
   - remappings maps material name (as seen on FBX) to existing material asset path
   - C# loads ModelImporter, sets materialImportMode to ImportViaMaterialDescription
   - For each remapping: loads target material via AssetDatabase.LoadAssetAtPath<Material>, calls importer.AddRemap(new AssetImporter.SourceAssetIdentifier(typeof(Material), materialName), targetMaterial)
   - Calls SaveAndReimport()
   - MenuItem: `VeilBreakers/Assets/Remap Materials`

9. `generate_material_auto_generate_script(fbx_path: str, texture_dir: str, shader_name: str = "Universal Render Pipeline/Lit") -> str`
   - Scans texture_dir for textures matching naming conventions: *_albedo.*, *_normal.*, *_metallic.*, *_roughness.*, *_ao.*, *_BaseColor.*, *_Normal.*, *_MetallicSmoothness.*, *_Occlusion.*
   - Creates Material with Shader.Find(shader_name)
   - Assigns textures to standard URP Lit properties: _BaseMap (albedo), _BumpMap (normal), _MetallicGlossMap (metallic), _OcclusionMap (AO)
   - Saves material to Assets/Materials/{material_name}.mat
   - Then remaps FBX to use this generated material
   - MenuItem: `VeilBreakers/Assets/Auto Generate Materials`

10. `generate_asmdef_script(name: str, root_dir: str, root_namespace: str = "", references: list[str] | None = None, platforms: list[str] | None = None, defines: list[str] | None = None, allow_unsafe: bool = False, auto_referenced: bool = True) -> str`
    - NOTE: This is NOT a C# script -- asmdef files are plain JSON. This function returns JSON content directly.
    - If root_namespace is empty, use name as root_namespace
    - references default to empty list
    - platforms (includePlatforms): empty list means all platforms
    - Output format matches Unity .asmdef spec:
    ```json
    {
        "name": "{name}",
        "rootNamespace": "{root_namespace}",
        "references": ["{ref1}", "{ref2}"],
        "includePlatforms": [],
        "excludePlatforms": [],
        "allowUnsafeCode": false,
        "overrideReferences": false,
        "precompiledReferences": [],
        "autoReferenced": true,
        "defineConstraints": ["{define1}"],
        "versionDefines": [],
        "noEngineReferences": false
    }
    ```
    - Also returns a companion function or the asmdef content directly (the handler in unity_server.py will write this via _write_to_unity to the specified root_dir path as {name}.asmdef)

11. `generate_preset_create_script(preset_name: str, source_asset_path: str, save_dir: str = "Assets/Editor/Presets") -> str`
    - C# loads the source asset/importer, creates `new Preset(source)`, saves via AssetDatabase.CreateAsset to `{save_dir}/{preset_name}.preset`
    - MenuItem: `VeilBreakers/Assets/Create Preset`

12. `generate_preset_apply_script(preset_path: str, target_path: str) -> str`
    - C# loads Preset from preset_path, loads target importer/object from target_path
    - Calls preset.ApplyTo(target) and target importer's SaveAndReimport()
    - MenuItem: `VeilBreakers/Assets/Apply Preset`

13. `generate_reference_scan_script(asset_path: str) -> str`
    - Finds the GUID of the target asset via AssetDatabase.AssetPathToGUID
    - Searches all project assets for references to that GUID (grep through .meta, .prefab, .unity, .mat, .asset files)
    - Alternative: uses AssetDatabase.GetDependencies in reverse (iterate all assets, check if target is in their dependencies)
    - Reports list of referencing assets in result JSON
    - MenuItem: `VeilBreakers/Assets/Scan References`

14. `generate_atomic_import_script(texture_paths: list[str], material_name: str, fbx_path: str, shader_name: str = "Universal Render Pipeline/Lit", remappings: dict[str, str] | None = None) -> str`

    - Step 4: Remap FBX materials to the material from step 2
    - Reports all changed_assets from all 4 steps
    - MenuItem: `VeilBreakers/Assets/Atomic Import`

@mcp.tool()
async def unity_assets(
    action: Literal[
        "move",              # EDIT-10 + IMP-01: Move asset (GUID-safe)
        "rename",            # EDIT-10 + IMP-01: Rename asset (GUID-safe)
        "delete",            # EDIT-10: Delete asset (with optional reference scan)
        "duplicate",         # EDIT-10: Duplicate asset
        "create_folder",     # EDIT-10: Create folder
        "configure_fbx",     # EDIT-12: FBX ModelImporter settings
        "configure_texture", # EDIT-13: TextureImporter settings
        "remap_materials",   # EDIT-14 + IMP-02: Material remapping on FBX
        "auto_materials",    # EDIT-14 + IMP-02: Auto-generate materials from textures
        "create_asmdef",     # EDIT-15: Assembly Definition creation
        "create_preset",     # PIPE-09: Create Unity Preset
        "apply_preset",      # PIPE-09: Apply Unity Preset
        "scan_references",   # IMP-01: Scan asset references
        "atomic_import",     # Combined import sequence
    ],
    # Asset operation params
    asset_path: str = "",
    new_path: str = "",
    new_name: str = "",
    safe_delete: bool = True,
    source_path: str = "",
    dest_path: str = "",
    folder_path: str = "",
    # FBX import params
    scale: float = 1.0,
    mesh_compression: str = "Off",
    animation_type: str = "None",
    import_animation: bool = False,
    normals_mode: str = "Import",
    import_blend_shapes: bool = True,
    optimize: bool = True,
    is_readable: bool = False,
    preset_type: str = "",
    # Texture import params
    max_size: int = 2048,
    srgb: bool = True,
    mipmap: bool = True,
    filter_mode: str = "Bilinear",
    wrap_mode: str = "Repeat",
    sprite_mode: str = "",
    platform_overrides: dict | None = None,
    auto_detect_srgb: bool = False,
    # Material remap params
    fbx_path: str = "",
