**CRITICAL: Create the `_resolve_selector_snippet` helper FIRST -- it is used by most generator functions below.**

`_resolve_selector_snippet(selector: dict | str) -> str` -- per user decision on deterministic selector schema:
- Accepts either a dict `{"by": "name"|"path"|"guid"|"regex", "value": str}` or a plain string (backward compat shorthand for `{"by": "name", "value": str}`)
- Returns a C# code snippet (multi-line string) that resolves a `GameObject target` variable from the selector.
- The generated C# snippet implements the **full selector chain** per the locked decision:
  - `by="name"`: `GameObject.Find(value)` -- primary lookup
  - `by="path"`: `GameObject.Find("/" + value)` for absolute paths (leading slash forces hierarchy-path lookup in Unity), OR `Transform.Find` chain for relative paths. This is the fallback when name alone is ambiguous (duplicated names, nested prefabs)
  - `by="guid"`: Uses `GlobalObjectId.TryParse` + `GlobalObjectId.GlobalObjectIdentifierToObjectSlow` for scene objects, or `AssetDatabase.GUIDToAssetPath` + `AssetDatabase.LoadAssetAtPath` for prefab assets. This handles variant chains safely since GUIDs are unique
  - `by="regex"`: Uses `System.Text.RegularExpressions.Regex.IsMatch` against all root GameObjects via `UnityEngine.SceneManagement.SceneManager.GetActiveScene().GetRootGameObjects()` then recurses children. Returns first match. Handles `FindObjectsOfType<GameObject>` as alternative when deep search needed
- The snippet must include null-check: if target is null after resolution, write error to vb_result.json and return early
- The snippet assigns the result to `GameObject target` so all downstream generator code uses `target` consistently

Create these 17 generator functions, each returning a complete C# editor script string:

1. `generate_prefab_create_script(name: str, prefab_type: str, save_dir: str, components: list[dict] | None = None) -> str`
   - If components is None, load from auto-wire profile JSON for prefab_type
   - C# creates GameObject, adds components from profile (using Undo.AddComponent), configures default properties via SerializedObject, calls PrefabUtility.SaveAsPrefabAsset, writes enhanced result JSON with changed_assets/warnings/validation_status
   - MenuItem: `VeilBreakers/Prefab/Create Prefab`
   - Script writes to: `Assets/Editor/Generated/Prefab/VeilBreakers_CreatePrefab.cs`

2. `generate_scaffold_prefab_script(name: str, prefab_type: str) -> str`
   - Creates ghost scaffold with CreatePrimitive(PrimitiveType.Capsule) as placeholder
   - Applies transparent material for ghost visual
   - Adds auto-wire components from profile
   - MenuItem: `VeilBreakers/Prefab/Create Scaffold`

3. `generate_prefab_variant_script(name: str, base_prefab_path: str, overrides: dict[str, str] | None = None) -> str`
   - Uses PrefabUtility.InstantiatePrefab(base) -> modify via SerializedObject -> SaveAsPrefabAsset (auto-detects variant)
   - MenuItem: `VeilBreakers/Prefab/Create Variant`

4. `generate_prefab_modify_script(prefab_path: str, modifications: list[dict]) -> str`
   - Each modification dict has: property, value, type (float/int/string/bool/object_ref/enum/color/vector3)
   - Uses AssetDatabase.LoadAssetAtPath, SerializedObject, FindProperty, type-appropriate value setter
   - MenuItem: `VeilBreakers/Prefab/Modify Prefab`

5. `generate_prefab_delete_script(prefab_path: str) -> str`
   - Uses AssetDatabase.DeleteAsset
   - MenuItem: `VeilBreakers/Prefab/Delete Prefab`

6. `generate_add_component_script(selector: dict | str, component_type: str, properties: list[dict] | None = None) -> str`
   - Uses `_resolve_selector_snippet(selector)` to find the target GameObject (NOT hardcoded `GameObject.Find`)
   - Adds component via Undo.AddComponent, optionally configures properties
   - MenuItem: `VeilBreakers/Prefab/Add Component`

7. `generate_remove_component_script(selector: dict | str, component_type: str) -> str`
   - Uses `_resolve_selector_snippet(selector)` to find the target GameObject
   - Finds component via GetComponent, removes via Undo.DestroyObjectImmediate
   - MenuItem: `VeilBreakers/Prefab/Remove Component`

8. `generate_configure_component_script(selector: dict | str, component_type: str, properties: list[dict]) -> str`
   - Uses `_resolve_selector_snippet(selector)` to find the target GameObject
   - Properties list: [{property: str, value: str, type: str}]
   - Uses SerializedObject/FindProperty with type-switch (floatValue/intValue/stringValue/boolValue/objectReferenceValue/enumValueIndex/colorValue/vector3Value)
   - MenuItem: `VeilBreakers/Prefab/Configure Component`

9. `generate_reflect_component_script(selector: dict | str, component_type: str) -> str`
   - Uses `_resolve_selector_snippet(selector)` to find the target GameObject
   - Creates SerializedObject, calls GetIterator/NextVisible(true) to enumerate all fields
   - Writes field name, type, propertyType, current value to result JSON
   - MenuItem: `VeilBreakers/Prefab/Reflect Component`

10. `generate_hierarchy_script(operation: str, **kwargs) -> str`
    - Operations: create_empty, rename, reparent, enable, disable, set_layer, set_tag, duplicate, delete
    - For operations that target an existing object (rename, reparent, enable, disable, set_layer, set_tag, duplicate, delete): uses `_resolve_selector_snippet(selector)` from kwargs to find the target. The `selector` kwarg accepts dict or string, same as component operations
    - Each operation uses appropriate Unity API with Undo registration
    - MenuItem: `VeilBreakers/Prefab/Hierarchy`

11. `generate_batch_configure_script(selector: dict, component_type: str, properties: list[dict]) -> str`
    - Selector dict: {by: "tag"|"layer"|"name_pattern", value: str}
    - Uses FindGameObjectsWithTag / FindObjectsOfType with layer filter / regex name match
    - Applies component + property config to all matches
    - MenuItem: `VeilBreakers/Prefab/Batch Configure`

12. `generate_variant_matrix_script(base_name: str, base_prefab_path: str, corruption_tiers: list[int], brands: list[str], output_dir: str = "Assets/Prefabs/Variants") -> str`
    - Wraps in AssetDatabase.StartAssetEditing/StopAssetEditing for performance
    - Nested loop: for each corruption tier x brand, creates a variant via InstantiatePrefab+SaveAsPrefabAsset
    - Naming: `{base_name}_C{tier}_{brand}`
    - MenuItem: `VeilBreakers/Prefab/Generate Variant Matrix`

13. `generate_joint_setup_script(selector: dict | str, joint_type: str, config: dict) -> str`
    - Uses `_resolve_selector_snippet(selector)` to find the target GameObject
    - joint_type: HingeJoint, SpringJoint, ConfigurableJoint, CharacterJoint, FixedJoint
    - config dict maps joint property names to values
    - Uses AddComponent<JointType>, then SerializedObject for config
    - Also supports PhysicMaterial creation: if config contains "physics_material", creates and assigns it
    - MenuItem: `VeilBreakers/Prefab/Setup Joint`

14. `generate_navmesh_setup_script(operation: str, selector: dict | str, **kwargs) -> str`
    - Uses `_resolve_selector_snippet(selector)` to find the target GameObject
    - Operations: add_obstacle (NavMeshObstacle with carve/size), add_link (NavMeshLink with start/end/width), configure_area (NavMesh.SetAreaCost), add_modifier (NavMeshModifier with area)
    - Uses com.unity.ai.navigation APIs: NavMeshLink (not OffMeshLink), NavMeshModifier
    - MenuItem: `VeilBreakers/Prefab/NavMesh Setup`

15. `generate_bone_socket_script(prefab_path: str, sockets: list[str] | None = None) -> str`
    - Default sockets (all 10): weapon_hand_R, weapon_hand_L, shield_hand_L, back_weapon, hip_L, hip_R, head, chest, spell_hand_R, spell_hand_L
    - Standard bone mappings dict: weapon_hand_R -> RightHand, etc.
    - Uses Animator.GetBoneTransform for Humanoid rigs, Transform.Find fallback for Generic
    - Creates empty GameObjects named Socket_{name} parented to bone transforms
    - MenuItem: `VeilBreakers/Prefab/Setup Bone Sockets`

16. `generate_validate_project_script() -> str`
    - Scans all prefabs via AssetDatabase.FindAssets("t:Prefab")
    - Reports missing components (GetComponents returns null entries), broken script references, missing materials
    - Checks tag/layer consistency against Constants.cs patterns
    - MenuItem: `VeilBreakers/Prefab/Validate Project Integrity`

17. `generate_job_script(operations: list[dict]) -> str`
    - **This implements the locked "batch scripting" decision** -- multiple different operation types in one compilation cycle
    - Each operation dict has: `action` (str -- one of "add_component", "remove_component", "configure", "hierarchy", "setup_joints"), plus the params for that action type (selector, component_type, properties, etc.)
    - Generates a SINGLE C# editor script that performs ALL operations sequentially in one `[MenuItem]` call
    - Wraps entire batch in `AssetDatabase.StartAssetEditing()` / `AssetDatabase.StopAssetEditing()` for performance (avoids per-operation reimport)
    - Uses `Undo.IncrementCurrentGroup()` at start so all operations can be undone as one atomic batch
    - For each operation, inlines the appropriate C# code (selector resolution, component add/remove/configure, hierarchy op, joint setup) -- reusing the same C# patterns from the individual generators
    - Collects all changed_assets across all operations into one result JSON
    - Reports per-operation status in a `steps` array within the result JSON: `[{step: 1, action: "add_component", status: "success"}, ...]`
    - If any operation fails, logs the error in the steps array but continues with remaining operations (best-effort)
    - MenuItem: `VeilBreakers/Prefab/Execute Job Script`
    - If operations list is empty, returns a minimal no-op script that writes a warning to vb_result.json
