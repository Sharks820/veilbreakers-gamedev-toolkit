    1. **Style Configs (STYLE_CONFIGS dict):**
       Five preset styles each defining: foundation (height, inset, material), walls (height_per_floor, thickness, material), floor_slab (thickness, material), roof (type=gabled|flat|domed|pointed|organic, overhang, pitch, material), windows (style, width, height, per_wall), door (style, width, height), details (list of decorative element names).

       - organic: curved walls (non-rectangular footprint hints), domed roof, vine details

    2. **BuildingSpec dataclass:**
       `BuildingSpec(footprint, floors, style, operations)` where operations is list of dicts like:
       `{"type": "box", "position": [x,y,z], "size": [w,h,d], "material": "stone", "role": "foundation"}`,
       `{"type": "cylinder", "position": [x,y,z], "radius": r, "height": h, "material": "stone", "role": "tower"}`,
       `{"type": "opening", "wall_index": n, "position": [x,y], "size": [w,h], "role": "window"}` (windows/doors are subtractive openings in walls).

    3. **evaluate_building_grammar(width, depth, floors, style, seed):**
       Evaluates grammar rules in order: foundation box -> per-floor wall boxes -> floor slabs between floors -> roof geometry -> window openings per wall -> door opening on front wall -> detail operations from style config. Returns BuildingSpec.

    4. **Specialized Templates:**
       - `generate_castle_spec(outer_size=40, keep_size=12, tower_count=4, seed=0)` -- curtain walls (4 wall segments), corner towers (cylinders with battlements), keep (central building via grammar), gatehouse (front opening with portcullis box)
       - `generate_tower_spec(radius=3.0, height=15.0, floors=3, seed=0)` -- cylindrical body, spiral stair placeholder, battlement ring at top
       - `generate_bridge_spec(span=20.0, width=5.0, arch_count=3, seed=0)` -- semicircular arches, road deck, railings, abutments
       - `generate_fortress_spec(size=60, wall_thickness=2.0, seed=0)` -- outer walls, corner towers, central keep, courtyard area, gatehouse

    5. **Ruins Damage (apply_ruins_damage):**
       - `apply_ruins_damage(spec, damage_level=0.5, seed=0) -> BuildingSpec`
       - damage_level [0, 1] controls destruction intensity
       - Randomly remove operations based on damage_level probability (roof goes first, then upper floors, walls last)
       - Add "debris" operations near removed elements (fallen stones, broken timbers)
       - Optionally add "vegetation" markers (overgrown ivy, moss patches) at higher damage levels

    6. **Interior Generation (generate_interior_layout):**
       - `generate_interior_layout(room_type, width, depth, height=3.0, seed=0) -> list[dict]`
       - Room type configs: tavern, throne_room, dungeon_cell, bedroom, kitchen, library, armory, chapel
       - Each config defines furniture items with placement rules (wall-hugging, centered, corner, etc.)
       - Collision avoidance: each placed item reserves a bounding box, subsequent items skip occupied areas
       - Returns list of placement dicts with type, position, rotation, scale

    7. **Modular Kit (MODULAR_CATALOG + generate_modular_pieces):**
       - `MODULAR_CATALOG` dict mapping piece names to specs: wall_straight, wall_corner, wall_t, floor, door_frame, window_frame, stairs, pillar

       - `generate_modular_pieces(cell_size=2.0, pieces=None) -> list[dict]` returns piece geometry specs scaled to cell_size. If pieces=None, generate all catalog entries.

  <action>
    Create `worldbuilding.py` with 5 handler functions:

    1. `handle_generate_building(params)` - Accepts: name, width (default 10), depth (default 8), floors (default 2), style (default "medieval"), seed (default 0). Calls evaluate_building_grammar(), converts BuildingSpec.operations to bmesh geometry:
       - "box" operations -> bmesh cube primitives at specified position/size
       - "cylinder" operations -> bmesh.ops.create_cone (with equal top/bottom radius)
       - "opening" operations -> construct wall faces around the opening (NO boolean subtract -- per research recommendation, direct bmesh face construction for window/door holes is more reliable than boolean)
       - Assign material slots per role (foundation, wall, roof, detail)
       - Returns dict with building metadata.

    2. `handle_generate_castle(params)` - Accepts: name, outer_size (default 40), keep_size (default 12), tower_count (default 4), style (default "fortress"), seed. Calls generate_castle_spec() to get composite BuildingSpec, then converts to mesh same as above. Castle is a single joined mesh object for clean export.

    3. `handle_generate_ruins(params)` - Accepts: name, source_building params OR existing_object, damage_level (default 0.5), seed. If source params provided, first generates building spec then applies apply_ruins_damage(). If existing_object, reads its spec from custom properties (or regenerates). Converts damaged spec to mesh, including debris objects. Uses separate mesh objects for main structure and debris collection.

    4. `handle_generate_interior(params)` - Accepts: name, room_type (default "tavern"), width, depth, height (default 3.0), seed. Calls generate_interior_layout(), creates simple placeholder geometry for each furniture item (boxes scaled to furniture dimensions). Each furniture item is a separate object parented to an empty for the room. Materials assigned per furniture type.

    5. `handle_generate_modular_kit(params)` - Accepts: name_prefix (default "ModKit"), cell_size (default 2.0), pieces (list of piece names or None for all). Calls generate_modular_pieces(), creates each piece as a separate mesh object. Origin at corner (0,0,0). Objects named "{prefix}_{piece_name}". Stores cell_size and connection_points as custom properties on each object for downstream snapping.
