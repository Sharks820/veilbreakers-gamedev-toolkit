  <action>
    Create `_scatter_engine.py` (pure-logic, NO bpy/bmesh imports) with:

    1. **Poisson Disk Sampling (poisson_disk_sample):**
       - `poisson_disk_sample(width, depth, min_distance, seed=0, max_attempts=30) -> list[tuple[float, float]]`
       - Implement Bridson's algorithm: active list, grid acceleration structure, O(n) time complexity
       - Grid cell size = min_distance / sqrt(2) for efficient neighbor checks
       - Returns list of 2D points within [0, width] x [0, depth] bounds

    2. **Biome Filter (biome_filter_points):**
       - `biome_filter_points(points, heightmap, slope_map, rules, terrain_size=100.0) -> list[dict]`
       - Rules format: list of dicts with vegetation_type, min_alt, max_alt, min_slope, max_slope, scale_range (min, max), density (probability of keeping point)
       - Maps each point's (x, y) to heightmap indices
       - Tests altitude and slope against each rule, assigns first matching vegetation type
       - Applies density probability (random per point using seed offset)
       - Returns placement dicts with position, vegetation_type, scale (random within range), rotation (random 0-360)

    3. **Context-Aware Scatter (context_scatter):**
       - `PROP_AFFINITY` dict: maps building types to weighted prop lists
         e.g., "tavern": [("barrel", 0.3), ("bench", 0.2), ("mug", 0.15), ("lantern", 0.1)],
         "dock": [("crate", 0.3), ("rope_coil", 0.2), ("barrel", 0.15), ("anchor", 0.1)],
         "blacksmith": [("anvil", 0.2), ("weapon_rack", 0.2), ("coal_pile", 0.15)],
         "graveyard": [("tombstone", 0.3), ("dead_tree", 0.15), ("lantern", 0.1)],
         Generic fallback: [("rock", 0.2), ("log", 0.15), ("mushroom", 0.1), ("bush", 0.1)]
       - `context_scatter(buildings, area_size, prop_density=0.3, seed=0) -> list[dict]`
       - Generate candidate positions via Poisson disk in area
       - For each position, find nearest building, compute affinity score based on distance
       - Select prop type by weighted random from affinity list (closer = higher affinity weight)
       - Exclude positions inside building footprints (rectangular exclusion zones)
       - Returns placement dicts with type, position, rotation, scale

    4. **Breakable Variants (generate_breakable_variants):**
       - `BREAKABLE_PROPS` dict mapping prop name to intact geometry spec and destruction config
       - `generate_breakable_variants(prop_type, seed=0) -> dict`
       - intact_spec: list of geometry operations (boxes, cylinders) describing the prop
       - destroyed_spec: fragment the intact geometry into N pieces (random plane cuts on bounding box), add small debris operations scattered around original position, darken material color
       - Standard props: barrel (cylinder -> 4-6 stave fragments), crate (box -> 4-8 plank fragments), pot (cylinder -> 3-5 shards), fence (box -> 2-3 broken sections), cart (complex -> 6-10 mixed fragments)

    **Part A: Create scatter handlers (environment_scatter.py)**

    Create `environment_scatter.py` with 3 handler functions:

    1. `handle_scatter_vegetation(params)` - Accepts: terrain_name, rules (biome rules list or use defaults), min_distance (default 3.0), seed (default 0), max_instances (default 5000). Extracts heightmap and slope data from terrain mesh, calls poisson_disk_sample() then biome_filter_points(), creates Blender geometry:
       - Use collection instances for performance (create template objects in a hidden collection, instance them at scatter positions)
       - Template objects: simple geometry (cone for tree, flat plane for grass, sphere for bush, box for rock)
       - Each vegetation_type gets its own template and material
       - Returns dict with: name, instance_count, vegetation_types (dict of type -> count), bounds

    2. `handle_scatter_props(params)` - Accepts: area_name, buildings (list of {type, position, footprint}), prop_density (default 0.3), seed (default 0). Calls context_scatter(), creates prop objects as collection instances:
       - Group props by type into separate instance collections
       - Position at terrain height if terrain exists (raycast or vertex lookup)
       - Returns dict with: name, prop_count, prop_types (dict of type -> count)

    3. `handle_create_breakable(params)` - Accepts: prop_type, position (default [0,0,0]), seed (default 0). Calls generate_breakable_variants(), creates two object variants:
       - Intact version: visible, uses intact geometry spec converted to bmesh
       - Destroyed version: hidden (viewport_display = False), fragments as separate objects in a collection, debris scattered nearby
       - Parent both to an empty named "{prop_type}_breakable"
       - Returns dict with: name, intact_vertex_count, fragment_count, debris_count
