    Create `_dungeon_gen.py` (pure-logic, NO bpy/bmesh imports, uses only stdlib + numpy) with:

    1. **Dataclasses:**
       - `Room(x, y, width, height, room_type="generic")` -- room_type in ("generic", "spawn", "boss", "treasure", "entrance", "exit")
       - `DungeonLayout(width, height, rooms, corridors, doors, spawn_points, loot_points, grid)` -- grid is 2D numpy int8 array (0=wall, 1=floor, 2=corridor, 3=door)
       - `CaveMap(width, height, grid, regions, largest_region)` -- grid is 2D numpy int8 (0=wall, 1=floor), regions is list of sets of (row,col), largest_region index
       - `TownLayout(width, height, districts, roads, building_plots, landmarks)` -- districts list of dicts with cells/type/center, roads set of (row,col), building_plots list of dicts with position/size/district

    2. **BSP Dungeon (generate_bsp_dungeon):**
       - `generate_bsp_dungeon(width=64, height=64, min_room_size=6, max_depth=5, seed=0) -> DungeonLayout`
       - Recursive BSP partition: split area horizontally or vertically (alternating or random based on aspect ratio), recurse until leaf is too small or max depth reached
       - Place rooms inside leaf nodes with random inset (room smaller than partition)
       - Connect sibling rooms with L-shaped corridors (horizontal then vertical segment)
       - Place doors where corridors meet room edges
       - Assign room types: first room = entrance, deepest room = boss, 1-2 random = treasure, remaining = generic
       - Place spawn points on floor cells in generic rooms, loot points in treasure rooms
       - Post-condition: `_verify_connectivity(layout)` using BFS flood-fill from entrance. If any room unreachable, add forced corridor. Raise ValueError if still disconnected.

    3. **Cave Generation (generate_cave_map):**
       - `generate_cave_map(width=64, height=64, fill_probability=0.45, iterations=5, seed=0) -> CaveMap`
       - Initialize random grid with fill_probability chance of wall per cell
       - Set border cells to wall
       - Apply cellular automata 4-5 rule for `iterations` rounds: cell becomes wall if >= 5 wall neighbors (including self), floor if < 4 wall neighbors
       - Flood-fill to identify connected floor regions
       - Keep only the largest connected region (set other floor cells to wall)
       - Return CaveMap with region data

    4. **Town Layout (generate_town_layout):**
       - `generate_town_layout(width=200, height=200, num_districts=6, seed=0) -> TownLayout`
       - Generate Voronoi-like districts: place `num_districts` seed points (random or semi-structured), assign each cell to nearest seed using Manhattan/Euclidean distance
       - Tag districts with types: civic (contains center), residential (largest area), commercial (adjacent to civic), industrial (edges)
       - Generate road network along district boundaries (cells adjacent to 2+ different districts)
       - Subdivide each district into building plots: rectangular subdivision with road-frontage preference
       - Place landmarks at district centers on road-adjacent cells

    5. **Helpers:**
       - `_verify_connectivity(layout) -> bool` -- BFS flood-fill from entrance room checking all rooms reachable
       - `_flood_fill(grid, start) -> set` -- returns set of connected floor cells from start position
       - `_count_neighbors(grid, x, y) -> int` -- count wall neighbors in 3x3 neighborhood

    Write RED tests first, confirm failure, then implement GREEN.
