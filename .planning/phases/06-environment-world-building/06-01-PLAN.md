  <action>
    Create `environment.py` with 6 handler functions following the established pattern (params dict in, result dict out):

    1. `handle_generate_terrain(params)` - Accepts: name, resolution (default 129), terrain_type (default "mountains"), scale (default 100.0), height_scale (default 20.0), seed (default 0), octaves, persistence, lacunarity, erosion (none/hydraulic/thermal/both), erosion_iterations. Calls _terrain_noise.generate_heightmap() to get heightmap array, optionally applies _terrain_erosion functions, then converts to Blender mesh using bmesh.ops.create_grid() with vertex Z from heightmap. Returns structured dict with terrain metadata.

    2. `handle_paint_terrain(params)` - Accepts: name (existing terrain object), biome_rules (list of dicts with name, material, min_alt, max_alt, min_slope, max_slope), height_scale. Uses _terrain_noise.compute_slope_map() on vertex heights to get slopes, _terrain_noise.compute_biome_assignments() for per-face material indices. Creates material slots on the mesh and assigns faces via bmesh. Default biome_rules use dark fantasy palette (snow, dark_rock, dead_grass, mud).

    3. `handle_carve_river(params)` - Accepts: terrain_name, source (grid coord), destination (grid coord), width (default 2), depth (default 0.05), seed. Extracts heightmap from mesh vertex Z values, calls _terrain_noise.carve_river_path(), updates vertex positions. Returns path length.

    4. `handle_generate_road(params)` - Accepts: terrain_name, waypoints (list of grid coords), width (default 3), grade_strength (default 0.8), seed. Similar to river but uses generate_road_path() and flattens terrain along route.

    5. `handle_create_water(params)` - Accepts: name (default "Water"), water_level, terrain_name (for sizing), width, depth, material_name. Creates a flat plane at water_level using bmesh, applies a water material with transparency. Returns water body metadata.

    6. `handle_export_heightmap(params)` - Accepts: terrain_name, filepath, flip_vertical (default True), unity_compat (default False). Extracts heightmap from mesh vertices, normalizes to [0,1], converts to 16-bit uint16, writes as RAW little-endian binary using struct.pack or numpy.tobytes(). If unity_compat=True, resizes to nearest power-of-two+1 dimension.

    Extract pure-logic validation functions (_validate_terrain_params, _export_heightmap_raw) for testing without Blender.

    Write test_environment_handlers.py testing pure-logic extraction functions. Handler functions that call bpy are tested via structured output verification (dict keys/types), not by running Blender.
