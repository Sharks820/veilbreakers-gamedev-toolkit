    Create animation.py following the established handler pattern (handle_*(params) -> dict).

    1. Implement helper functions:
       - _validate_animation_params(params): Extracts and validates object_name, checks armature type, returns (armature_obj, params). Raises ValueError with helpful message on failure.
       - _get_armature_def_bones(armature_obj): Returns sorted list of DEF-prefixed bone names from armature's pose bones.
       - _resolve_bone_name(armature_obj, bone_name): Checks if bone exists, tries with/without DEF- prefix. Returns resolved name or raises ValueError.
       - _apply_keyframes_to_action(armature_obj, action_name, keyframes, use_cyclic=True): Creates bpy Action, creates fcurves grouped by (bone, channel, axis), bulk-inserts keyframes, adds CYCLES modifier if use_cyclic. Returns {"action_name", "fcurve_count", "frame_range"}.
       - _add_contact_markers(action, contact_frames): Adds pose_markers to action at each frame.

    2. handle_generate_walk(params) -> dict (ANIM-01):
       Params: object_name, gait (default "biped"), speed (default "walk"), frame_count (default 24).
       - Validate params and armature
       - Get DEF bone names from armature
       - Call get_gait_config(gait, speed, frame_count, bone_names)
       - Call generate_cycle_keyframes(config)
       - Apply keyframes with _apply_keyframes_to_action (use_cyclic=True)
       - Add contact markers from config["contact_frames"]
       - Return structured result dict

    3. handle_generate_fly(params) -> dict (ANIM-02):
       Params: object_name, frequency (default 2.0), amplitude (default 0.8), glide_ratio (default 0.3), frame_count (default 24).
       - Get FLY_HOVER_CONFIG, override frequency/amplitude/glide_ratio params
       - Apply keyframes with cyclic
       - Return result dict

    4. handle_generate_idle(params) -> dict (ANIM-03):
       Params: object_name, frame_count (default 48), breathing_intensity (default 1.0).
       - Get IDLE_CONFIG, scale amplitudes by breathing_intensity
       - Apply keyframes with cyclic
       - Return result dict

    Write tests for the pure-logic validation functions (_validate_animation_params input parsing, _get_armature_def_bones output format). Test class: TestAnimationValidation, TestWalkParams, TestFlyParams, TestIdleParams.

    IMPORTANT: Check pose_bone.rotation_mode before generating keyframes. If QUATERNION, use "rotation_quaternion" channel with axis 0-3. If Euler, use "rotation_euler" with axis 0-2. The _apply_keyframes_to_action helper must handle this.
