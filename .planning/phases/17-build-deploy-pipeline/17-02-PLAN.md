       - Environment variables: UNITY_LICENSE, UNITY_EMAIL, UNITY_PASSWORD from secrets
       - Test job (if run_tests): uses game-ci/unity-test-runner@v4, testMode: all, with LFS checkout and Library caching
       - Build job (needs: test if run_tests, else standalone): strategy.matrix.targetPlatform with all platforms, uses game-ci/unity-builder@v4, upload-artifact@v4
       - actions/checkout@v4 with lfs: true
       - actions/cache@v3 for Library folder with hash key
       - Use Python string building for YAML (NOT yaml library -- keep it simple, match template conventions)

    2. `generate_gitlab_ci_config(unity_version="6000.0.0f1", platforms=None, namespace="") -> str` (BUILD-03):
       - Returns YAML string (NOT C#).
       - Default platforms same as above
       - Image: unityci/editor:ubuntu-{unity_version}-{platform}-3
       - Stages: test, build
       - Test job: stage test, script runs Unity test runner CLI
       - Build jobs: one per platform, stage build, script runs Unity batch mode build
       - Variables: UNITY_LICENSE_CONTENT (from CI/CD variable)
       - Before_script: license activation steps
       - Artifacts paths for build output
       - Cache Library folder

    3. `generate_version_management_script(version="1.0.0", auto_increment="patch", update_android=True, update_ios=True, namespace="") -> str` (BUILD-04):
       - Returns C# editor script
       - [MenuItem("VeilBreakers/Build/Bump Version")]
       - Parse current version from PlayerSettings.bundleVersion using string.Split('.')
       - Increment specified component (major/minor/patch)
       - Set PlayerSettings.bundleVersion = new version
       - If update_android: PlayerSettings.Android.bundleVersionCode++ (integer increment)
       - If update_ios: PlayerSettings.iOS.buildNumber = new version string
       - Log the version change
       - Write result to Temp/vb_result.json
       - Required usings: UnityEditor, UnityEngine, System.IO

    4. `generate_changelog(project_name="VeilBreakers", version="1.0.0", namespace="") -> str` (BUILD-04):
       - Returns C# editor script that reads git log and generates CHANGELOG.md
       - [MenuItem("VeilBreakers/Build/Generate Changelog")]
       - Uses System.Diagnostics.Process to run `git log --pretty=format:"%h %s" --no-merges HEAD...{previous_tag}` where previous_tag is discovered via `git describe --tags --abbrev=0`
       - Formats output as markdown with ## [version] header, date, and bullet-pointed commits
       - Groups commits by prefix (feat:, fix:, docs:, etc.) if conventional commits are used
       - Writes CHANGELOG.md to project root
       - Writes result to Temp/vb_result.json with commit_count
       - Required usings: UnityEditor, UnityEngine, System, System.Diagnostics, System.IO, System.Text

    5. `generate_store_metadata(game_title="VeilBreakers", genre="Action RPG", has_iap=False, has_ads=False, collects_data=False, namespace="") -> str` (ACC-02):
       - Returns a MARKDOWN string (NOT C#) with three sections:
       - Section 1 -- Store Description: game_title, genre, 5 feature bullet points (placeholder), system requirements, age rating note
       - Section 2 -- Content Rating Questionnaire: ESRB/PEGI/IARC answers template for a dark fantasy action RPG. Pre-fill typical answers: Violence=Frequent (fantasy), Blood=Stylized, Language=Mild, Substances=None, Gambling=None, Sexual Content=None. Add "REVIEW BEFORE SUBMISSION" disclaimer.
       - Section 3 -- Privacy Policy Template: Standard privacy policy markdown. Sections: Information Collected (based on collects_data flag), How Information Is Used, Third-Party Services (based on has_ads flag), In-App Purchases (based on has_iap flag), Children's Privacy (COPPA note), Contact Information (placeholder). Add "THIS IS A TEMPLATE -- CONSULT A LAWYER BEFORE USE" disclaimer.
       - Section 4 -- Screenshot Specifications: Required screenshot sizes per platform (iOS: 6.7", 5.5"; Android: phone + 7" tablet + 10" tablet; Steam: 1920x1080 minimum)
