Established compound tool wiring rules:
1. Each tool has a Literal action parameter listing all actions
2. Tool docstring describes all actions with their parameters
3. Each action dispatches to the corresponding template generator
4. C# outputs: call _write_to_unity, return JSON with file_path + next_steps
5. Non-C# outputs (Git LFS, Blender Python): return result directly
6. All user-provided strings sanitized via _sanitize_cs_identifier before passing to generators
