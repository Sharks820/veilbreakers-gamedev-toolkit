```python
def _sanitize_cs_string(value: str) -> str:
    """Escape a value for safe embedding inside a C# string literal."""
    value = value.replace("\\", "\\\\")
    value = value.replace('"', '\\"')
    value = value.replace("\n", "\\n")
    value = value.replace("\r", "\\r")
    return value

def _sanitize_cs_identifier(value: str) -> str:
    """Sanitize a value for use as a C# identifier (class name, method name)."""
    return re.sub(r"[^a-zA-Z0-9_]", "", value)
```

VeilBreakers conventions:
- Category folder: Assets/Data/{Category}/ with PascalCase naming
- _camelCase private fields, PascalCase public/properties
- VeilBreakers.Data namespace for SO definitions
- VeilBreakers.Editor namespace for editor tools
- VB result output: Temp/vb_result.json
- Localization keys: UI.MainMenu.StartGame, Combat.Brand.Iron pattern
- Default locale: English (en)
- SO complements JSON (not replaces) -- for data needing Unity references

Existing VeilBreakers patterns:
- HeroDisplayConfig ScriptableObject at Assets/Resources/CharacterSelect/HeroDisplayConfigs/
- GameDatabase.cs async JSON loader at Assets/Scripts/Core/
