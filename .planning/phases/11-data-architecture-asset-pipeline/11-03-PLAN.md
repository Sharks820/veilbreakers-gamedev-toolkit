```python
# Lab color space technique:
# 1. Convert RGB to luminance (ITU-R BT.601 weights: 0.299, 0.587, 0.114)
```

Module docstring listing export: `delight_albedo`.

```python
def delight_albedo(
    image_path: str,
    output_path: str,
    blur_radius_pct: float = 0.12,
    strength: float = 0.75,
    clamp_min: float = 0.0,
    clamp_max: float = 1.0,
) -> dict:
    """Remove baked-in lighting from an albedo texture.

    Uses luminance channel analysis with Gaussian blur to estimate
    and divide out lighting, producing a flat-lit albedo suitable
    for PBR workflows.

    Args:
        image_path: Path to input albedo texture.
        output_path: Path to save de-lit result.
        blur_radius_pct: Gaussian blur radius as fraction of image dimension (0.0-1.0).
        strength: Blend strength between original and corrected (0.0-1.0).
        clamp_min: Minimum output value clamp.
        clamp_max: Maximum output value clamp.

    Returns:
        Dict with input, output, blur_radius, strength, mean_luminance_before,
        mean_luminance_after, correction_applied.
    """
```

Implementation (from 11-RESEARCH.md algorithm):
1. Open image with PIL, convert to RGB
2. Convert to numpy float32 array normalized to 0-1
3. Calculate luminance: `0.299 * R + 0.587 * G + 0.114 * B` (ITU-R BT.601)
4. Clip luminance minimum to 0.01 to avoid division by zero
5. Calculate blur radius: `int(max(width, height) * blur_radius_pct)`
6. Create luminance PIL Image, apply GaussianBlur with calculated radius
7. Convert blurred back to numpy, clip to 0.01 min
8. Calculate target luminance: `np.mean(luminance)`
9. Calculate correction factor: `target_luminance / lighting_estimate`
10. Apply per-channel: `corrected = original * correction`, blend with strength
11. Clamp to [clamp_min*255, clamp_max*255], convert to uint8
12. Save result via PIL
13. Return metadata dict

Handle edge cases:
- Very small images (< 16px): set blur_radius minimum to 1
- All-black or all-white images: skip correction, return original
- Missing numpy: raise ImportError with helpful message

```python
def validate_palette(
    image_path: str,
    rules: dict | None = None,  # Override PALETTE_RULES
    sample_pixels: int = 10000,  # Number of pixels to sample (for performance)
) -> dict:
    """Validate an albedo texture against dark fantasy palette rules.

    Returns dict with:
        passed: bool,
        issues: list of {rule, value, threshold, severity},
        stats: {mean_saturation, mean_value, cool_ratio, warm_ratio, ...}
    """
```

Implementation:
1. Open image, convert to RGB
2. Sample random pixels (or all if small enough)
3. Convert sampled pixels to HSV color space
4. Check saturation cap: mean saturation must be <= rules["saturation_cap"]
5. Check value range: pixels must mostly fall within value_range min/max
6. Check color temperature: classify pixels as warm (hue 0-60 or 300-360) vs cool (hue 120-240), ratio of cool must be >= cool_bias_target
7. Report issues with severity (error/warning)
