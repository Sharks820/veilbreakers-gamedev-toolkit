1. **Auto-generate LOD chains** (mesh_enhance.py, new function):
   - Add `auto_generate_lod_chain(obj_name: str, asset_type: str) -> dict`:
     - asset_type determines LOD ratios:
       - "tree": [1.0, 0.4, 0.06, 0.001] (LOD0 full, LOD1 40%, LOD2 6% cards, LOD3 billboard)
       - "building": [1.0, 0.5, 0.15] (LOD0 full, LOD1 50%, LOD2 15%)
       - "rock": [1.0, 0.3, 0.05] (LOD0 full, LOD1 30%, LOD2 5%)

     - For each LOD level: duplicate object, apply decimate modifier (ratio from above), rename to "{obj_name}_LOD{n}"
     - For tree LOD3: replace with billboard cross (2 quads)
     - Parent all LODs under an empty named "{obj_name}_LOD_group"
     - Return {lod_count, lod_objects: [{name, tri_count, ratio}]}

2. **GPU instancing setup** (environment_scatter.py):
   - In the multi-pass scatter (from plan 02), convert repeated meshes to instances:
     - After Pass 1 (trees): group identical tree species, convert copies to Blender instances (linked duplicates)
     - After Pass 2 (grass): all grass cards of same biome type -> instances
     - After Pass 3 (rocks): group by size category, instance within category
   - Set `instance_type = 'COLLECTION'` on parent empties for Blender Collection instancing
   - Tag each instance group with custom property `gpu_instancing = True` for Unity export
   - Expected draw call reduction: 100+ individual meshes -> 5-10 instanced groups

3. **Performance budget enforcement** (blender_server.py, add to aaa_verify action):
   - After running aaa_verify_map, also check performance:
     - Count total scene triangles (sum all mesh objects' face count * 2 for tris estimate)
     - Count unique materials (proxy for draw calls: each material = potential draw call)
     - Check per-category budgets from research spec table
     - Return {total_tris, tri_budget_ok (< 2M), estimated_draw_calls, draw_call_budget_ok (< 500), per_category: [{type, tris, budget, within_budget}]}
   - Add `performance_check` sub-action to asset_pipeline that ONLY runs the budget check (no screenshots)

4. **Topology grade enforcement** (mesh_enhance.py):
   - Add `enforce_topology_grade(obj_name: str, min_grade: str = "B+") -> dict`:
     - Run existing topology grading (game_check)
     - If grade < min_grade: attempt auto-repair (remove doubles, fix normals, dissolve degenerate faces)
     - Re-grade after repair
     - If still below: log warning with specific issues
     - Return {original_grade, repaired, final_grade, meets_minimum, issues}
   - Wire into compose_map: after LOD generation, enforce B+ on all LOD0 meshes
