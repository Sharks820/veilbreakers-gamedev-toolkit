    Create rigging.py following the established handler pattern from mesh.py:

    1. **Module docstring** listing all handlers and their requirement IDs (RIG-01, RIG-02, RIG-03).

    2. **Imports:** bpy, bmesh, mathutils, math, json, re, get_3d_context_override from ._context, template dicts/functions from .rigging_templates.

    3. **Pure-logic functions (testable without Blender):**

    `_analyze_proportions(bbox_dims: tuple[float,float,float], vertex_count: int, has_symmetry: bool) -> dict`:
    - Computes aspect_ratio = height / max(width, 0.001)
    - Template recommendation logic:
      - aspect > 2.5 and has_symmetry -> "humanoid" (0.8 confidence)
      - aspect < 0.5 and width > height -> "serpent" (0.7)
      - 0.3 < aspect < 0.8 and depth > width * 0.8 -> "quadruped" (0.7)
      - 0.8 < aspect < 2.5 and has_symmetry -> check for insect/arachnid indicators
      - fallback -> "amorphous" (0.4)
    - Returns: {"aspect_ratio": float, "recommended_template": str, "confidence": float, "has_symmetry": bool, "vertex_count": int, "all_candidates": list}

    `_validate_custom_rig_config(limb_types: list[str]) -> dict`:
    - Validates each limb type exists in LIMB_LIBRARY
    - Checks for conflicts (e.g., multiple spine types)
    - Returns: {"valid": bool, "errors": list[str], "limb_count": int, "bone_estimate": int}

    4. **Blender-dependent handlers:**

    `handle_analyze_for_rigging(params: dict) -> dict` (RIG-01):
    - Gets mesh object by name from params["object_name"]
    - Creates bmesh, computes bounding box dims, vertex count
    - Checks X-axis symmetry by comparing mirrored vertex positions
    - Calls _analyze_proportions with extracted data
    - Returns analysis result with template recommendation

    `handle_apply_rig_template(params: dict) -> dict` (RIG-02):
    - Gets mesh object, template name from params
    - Looks up template in TEMPLATE_CATALOG
    - Creates new armature object
    - Calls _create_template_bones(armature_obj, template_bones)
    - Calls _generate_rig(armature_obj) to produce control rig
    - Calls _fix_deform_hierarchy(rig_obj) for game export
    - Returns: {"rig_name": str, "bone_count": int, "template": str, "def_bone_count": int}

    `handle_build_custom_rig(params: dict) -> dict` (RIG-03):
    - Gets limb_types list from params
    - Validates config via _validate_custom_rig_config
    - Merges bone dicts from LIMB_LIBRARY functions
    - Always includes a spine as root
    - Creates armature, applies merged bones, generates rig
    - Returns: {"rig_name": str, "bone_count": int, "limbs_used": list, "def_bone_count": int}
