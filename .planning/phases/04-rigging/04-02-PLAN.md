    4. **Pure-logic functions:**

    `_compute_rig_grade(unweighted_pct, non_normalized, symmetry_issues, roll_issues) -> str`:
    - F: unweighted_pct > 10 or non_normalized > 100
    - D: unweighted_pct > 5 or non_normalized > 50 or symmetry_issues > 5
    - C: unweighted_pct > 1 or non_normalized > 10 or symmetry_issues > 2
    - B: unweighted_pct > 0 or non_normalized > 0 or roll_issues > 2
    - A: everything at 0

    `_validate_rig_report(vertex_count, vertex_group_names, unweighted_vertex_indices, weight_sums, bone_names, bone_rolls, bone_parents) -> dict`:
    - Computes unweighted_pct from unweighted_vertex_indices
    - Finds non-normalized vertices (abs(total - 1.0) > 0.01)
    - Checks L/R bone symmetry (finds left bones without right counterparts)
    - Checks bone roll consistency (L bone roll should be negative of R bone roll)
    - Returns: {"vertex_count", "bone_count", "unweighted_vertices", "unweighted_percentage", "non_normalized_vertices", "symmetry_issues", "roll_issues", "issues" (list of strings), "grade"}

    `_validate_weight_fix_params(operation: str, params: dict) -> dict`:
    - Valid operations: "normalize", "clean_zeros", "smooth", "mirror"
    - "mirror" requires "direction" in params (one of "left_to_right", "right_to_left")
    - "smooth" optionally takes "factor" (0.0-1.0) and "repeat" (1-10)
    - "clean_zeros" optionally takes "threshold" (0.0-0.1)
    - Returns: {"valid": bool, "errors": list[str], "operation": str}

    5. **Blender-dependent handlers:**

    `handle_auto_weight(params: dict) -> dict` (RIG-07):
    - Gets mesh object and armature object by name
    - Selects mesh, makes armature active
    - Uses temp_override + bpy.ops.object.parent_set(type='ARMATURE_AUTO', xmirror=True)
    - Handles "Bone Heat Weighting Failed" error with fallback to ARMATURE_ENVELOPE
    - Returns: {"vertex_group_count": int, "vertex_groups": list[str], "method": "auto"|"envelope", "mesh": str, "armature": str}

    `handle_test_deformation(params: dict) -> dict` (RIG-08):
    - Gets rig object by name, optional pose_names filter
    - For each pose in DEFORMATION_POSES (or filtered subset):
      - Sets pose bone rotations from preset
      - Renders frame via existing viewport infrastructure
    - Resets to rest pose after
    - Calls handle_render_contact_sheet to generate contact sheet of all 8 poses
    - Returns: {"poses_tested": int, "pose_names": list[str], "contact_sheet": str (base64 or path)}

    `handle_validate_rig(params: dict) -> dict` (RIG-09):
    - Gets mesh and armature objects
    - Extracts: vertex count, vertex group names, unweighted vertex indices (vertices with no weight > 0.01 in any group)
    - Computes weight sums per vertex
    - Gets bone names, rolls, parents from armature
    - Calls _validate_rig_report with extracted data
    - Returns validation report dict

    `handle_fix_weights(params: dict) -> dict` (RIG-10):
    - Gets mesh object, operation type from params
    - Validates via _validate_weight_fix_params
    - Executes appropriate bpy.ops with temp_override:
      - "normalize": bpy.ops.object.vertex_group_normalize_all()
      - "clean_zeros": bpy.ops.object.vertex_group_clean(group_select_mode='ALL', limit=threshold)
      - "smooth": bpy.ops.object.vertex_group_smooth(group_select_mode='ALL', factor=factor, repeat=repeat)
      - "mirror": bpy.ops.object.vertex_group_mirror(use_topology=False)
    - Returns: {"operation": str, "mesh": str, "status": "success"}
