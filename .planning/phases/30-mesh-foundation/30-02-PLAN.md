## Success Criteria from ROADMAP (what must be TRUE)

1. Every furniture generator (table, chair, chest, barrel, shelf, bed, bookshelf, crate) produces mesh with >500 vertices, proper edge flow at joints, and contact sheet shows silhouette distinguishable from primitive shapes at 4 viewing angles
2. All 267 generators use `random.Random(seed)` exclusively — zero instances of global random state — and identical seeds produce byte-identical MeshSpec output across runs
3. Smart material auto-assignment maps each generator category to the appropriate procedural material preset (wood for furniture, stone for architecture, metal for weapons) with roughness driven by noise texture nodes, never a single float value
4. LOD chain generation produces 3-4 levels per asset type using existing `lod_pipeline.py` presets, with silhouette preservation >85% measured by `compute_silhouette_importance()`, and scene budget validator enforces per-room (50K-150K tris) and per-block (200K-500K tris) budgets
5. Post-boolean cleanup pipeline (remove doubles at 0.0001, recalculate normals, fix non-manifold, fill holes up to 8 sides) runs automatically after every boolean operation, and no exported mesh contains non-manifold edges

## Plans

### Plan 2.1: Wire Procedural Materials as Default (MESH-02)

**Goal:** Every generated mesh automatically receives an appropriate procedural material instead of flat single-color Principled BSDF.

**Actions:**
1. Create material category mapping in `_mesh_bridge.py`:
   - FURNITURE generators → `rough_timber` or `polished_wood` (from procedural_materials.py)
   - WEAPON generators → `rusted_iron` or `polished_steel`
   - ARCHITECTURE generators → `rough_stone_wall` or `brick_wall`
   - VEGETATION generators → `bark` (trunks), `leaf` (canopy)
   - CONTAINER generators → `aged_wood` (crates), `chain_metal` (bands)
2. Modify `mesh_from_spec()` to auto-assign material after object creation
3. Fix `texture_quality.py` SMART_MATERIAL_PRESETS to use noise-driven roughness chains instead of scalar values (currently scalar at wrinkle_maps.py:434,437)
4. Validate: no generated object uses basic `handle_material_create` flat-color path

**Success Criteria:**
- Category → material mapping defined for all 21 generator categories
- `mesh_from_spec()` auto-applies procedural material
- Contact sheet of 5 generators shows visible roughness variation (not uniform plastic)
- Zero flat-color Principled BSDF materials on generated objects

**Verification:** Blender viewport render of table, chair, barrel, rock, sword — each must show material-appropriate surface quality

### Plan 2.2: Enforce Seed-Based RNG (MESH-14)

**Goal:** All 267 generators use `random.Random(seed)` exclusively. Zero global random state.

**Actions:**
1. Grep `procedural_meshes.py` for all random usage patterns:
   - `_rng.seed(N)` — GLOBAL STATE, must fix
   - `random.random()` — GLOBAL STATE, must fix
   - `random.Random(seed)` — CORRECT, keep
2. For each violation, replace with `rng = random.Random(seed)` instance pattern
3. Ensure all `generate_*` functions accept `seed` parameter
4. Create determinism test: call generator twice with same seed, assert MeshSpec equality

**Success Criteria:**
- `grep -c "_rng.seed\|random\.random()" procedural_meshes.py` returns 0
- Determinism test passes for 10 representative generators
- All 267 functions accept `seed` parameter

**Verification:** pytest determinism test

### Plan 2.3: Add Edge Loops + Bevel to Key Generators (MESH-01, MESH-06)

**Goal:** Upgrade BASIC-tier generators to catch light properly via edge loops and bevels.

**Actions:**
1. Create `_enhance_mesh_detail()` function in procedural_meshes.py:
   - Accept MeshSpec, detect sharp edges (angle > 30°)
   - Add supporting edge loops near sharp edges (bmesh.ops.subdivide_edges)
   - Apply bevel to hard edges (bmesh.ops.bevel, segments=2, offset=0.02)
   - Increase vertex count to meet minimum thresholds
2. Apply to furniture generators: table, chair, chest, door, shelf, bed, bookshelf, crate
3. Validate vertex count meets minimums: furniture >500, props >200

**Success Criteria:**
- `_enhance_mesh_detail()` function created and tested
- 8 furniture generators produce >500 vertices each
- Contact sheet comparison (before/after) shows visible improvement in edge definition
- Zero non-manifold edges in enhanced output

**Verification:** Contact sheet before/after comparison in Blender viewport

### Plan 2.4: Add Furniture LOD Preset + Scene Budget Validator (MESH-11)

**Goal:** Add missing furniture LOD tier and implement scene-level polygon budget enforcement.

**Actions:**
1. Add to LOD_PRESETS in `lod_pipeline.py`:
   ```python
   "furniture": {
       "ratios": [1.0, 0.5, 0.25],
       "screen_percentages": [1.0, 0.3, 0.1],
       "min_tris": [200, 100, 50],
   }
   ```
2. Create `SceneBudgetValidator` class (new file or extend `lod_pipeline.py`):
   - Sum all visible object triangle counts in current scene
   - Compare against budgets: per-room (50K-150K), per-block (200K-500K), frame (2M-6M at 60fps)
   - Return structured report: {total_tris, budget, utilization_pct, over_budget: bool, recommendations: []}
   - Recommendations: suggest LOD culling, material consolidation, mesh simplification
3. Wire into `blender_mesh action=game_check` as additional validation step

**Success Criteria:**
- `furniture` LOD preset added and generates correct LOD chain
- Scene budget validator reports accurate triangle counts
- Validator flags scenes exceeding per-room budget
- Integrated with game_check action

**Verification:** Create test scene with known polygon count, validate report accuracy

### Plan 2.5: Boolean Cleanup Pipeline (MESH-15)

**Goal:** Automatic mesh cleanup after boolean operations.

**Actions:**
1. Create `post_boolean_cleanup()` in `_mesh_bridge.py`:
   - Remove doubles (merge distance 0.0001)
   - Recalculate normals (weighted by face angles)
   - Fix non-manifold edges (boundary detection + fill)
   - Fill holes (up to 8 sides)
   - Return cleaned MeshSpec with cleanup report
2. Wire into all boolean operation handlers in mesh_handlers.py
3. Add cleanup validation to export pipeline (reject non-manifold on export)

**Success Criteria:**
- `post_boolean_cleanup()` function created
- Boolean operation followed by cleanup produces zero non-manifold edges
- Export pipeline rejects meshes with non-manifold geometry
- Cleanup report includes: doubles_removed, normals_fixed, holes_filled counts

**Verification:** Create mesh with known boolean artifacts, run cleanup, validate clean output
