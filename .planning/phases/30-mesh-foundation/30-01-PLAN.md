# Phase 30 (P0): Mesh Foundation - Plan 1

**Created:** 2026-03-30
**Plan Iteration:** 1 (initial)

## Phase Boundary from ROADMAP

Execute 20+ parametric mesh generators (tables, chairs, barrels, chests, shelves, beds, rocks, trees, bushes) replacing all placeholder primitives. Create material presets with roughness variation (never single float - always use roughness texture maps with procedural noise and curvature-based wear maps). Implement LOD presets per asset type (hero characters: 40K-60K/20K-30K/8K-15K/2K-5K; building modular pieces: 2K-8K/1K-4K/500-2K/cull; vegetation with billboard fallback). Add boolean cleanup pipeline and silhouette validation. Ensure scene-level polygon budget validator (50K-150K per room interior, 200K-500K per town block).

## Success Criteria from ROADMAP

1. 20+ parametric mesh generators exist and produce real mesh geometry (no cubes, cones, spheres)
2. Material presets include roughness variation (texture maps, not single float values)
3. LOD presets defined per asset type with silhouette preservation
4. Boolean cleanup pipeline implemented
5. Scene-level budget validator exists
6. All generators pass quality gate tests

## Research Summary (from v4.0 PITFALLS.md)

**Critical Pitfalls Addressed:**
- Pitfall 1: Placeholder primitives masquerading as AAA assets
  - **Solution:** Build parametric mesh generators FIRST. Enforce "no primitives in production" rule.
- Pitfall 2: Uniform roughness makes everything look like plastic
  - **Solution:** Always use roughness texture maps with procedural noise and curvature-based wear maps.
- Pitfall 9: bpy in pure-logic code
  - **Solution:** Pure-logic / bpy-guarded split enforced. All generators return MeshSpec dicts.

**Existing Codebase:**
- 127+ procedural mesh generators across 21 categories in `procedural_meshes.py`
- Pure-logic/bpy-guarded split: generators return MeshSpec dicts, only `_mesh_bridge.py` touches bpy/bmesh
- xatlas for UV unwrapping
- pymeshlab for high-quality remeshing/decimation
- LOD pipeline with per-asset-type presets

## Plans

### Plan 1.1: Audit Existing Generators (MESH-01.1)

**Goal:** Identify all generators producing primitives and create upgrade plan.

**Actions:**
1. Scan all generators in `procedural_meshes.py` for primitive output
2. Create gap list: generator name, current output, target output type
3. Prioritize by visual impact (furniture > props > vegetation)

**Success Criteria:**
- All 127+ generators audited
- Gap analysis document created (X-01 detailed gap report)
- Upgrade priority established

### Plan 1.2: Create Parametric Mesh Generator Library (MESH-01.2)

**Goal:** Build 20+ parametric generators replacing all primitives.

**Generators to Create:**

**Furniture (high priority):**
1. `parametric_table_generator()` - 4 legs, rectangular/circular top, thickness parametric
2. `parametric_chair_generator()` - 4 legs, backrest optional, seat/backrest options
3. `parametric_barrel_generator()` - Stave + bands, varying heights/diameters
4. `parametric_chest_generator()` - Rectangular body, hinged lid, lock mechanism
5. `parametric_shelf_generator()` - Adjustable compartments, back panel option
6. `parametric_bed_generator()` - Frame + mattress, headboard optional

**Storage (medium priority):**
7. `parametric_crates_generator()` - Wooden slats, varying sizes
8. `parametric_barrel_rack_generator()` - Vertical stacking slots

**Vegetation (high priority):**
9. `parametric_tree_generator()` - Trunk + branch system, variable foliage density
10. `parametric_bush_generator()` - Leaf clumps, branch variation

**Environment (low priority):**
11. `parametric_rock_generator()` - Erosion-based surface detail
12. `parametric_ground_clutter_generator()` - Small debris, twigs, pebbles

**Success Criteria:**
- 20+ parametric generators implemented
- All return MeshSpec dicts (pure logic)
- All use seed-based RNG (random.Random(seed))
- All pass basic topology validation

### Plan 1.3: Implement Material Preset System (MESH-01.3)

**Goal:** Create PBR material presets with roughness variation.

**Actions:**
1. Create `MaterialPresets` class with style presets:
   - Dark fantasy (IRON: iron-clad, dark steel)
   - Wood (oak, pine, weathered)
   - Stone (granite, sandstone, marble)
2. Implement roughness texture map generator:
   - Procedural noise (Perlin-like) for variation
   - Curvature-based wear (high edges = worn)
3. Create material assignment registry
4. Add material validation (check single float assignment)

**Success Criteria:**
- Material presets class created
- Roughness texture maps generated
- Generator → material assignment flow exists
- Single float roughness rejected (enforced)

### Plan 1.4: Implement LOD Preset System (MESH-01.4)

**Goal:** Define per-asset-type LOD budgets with silhouette preservation.

**Actions:**
1. Create `LODPresets` class with asset type definitions:
   - HERO_CHARACTER: [60K, 30K, 15K, 5K, 2K]
   - HERO_WEAPON: [20K, 10K, 5K, 2K]
   - BUILDING_PIECE: [8K, 4K, 2K, 1K, cull]
   - FURNITURE: [4K, 2K, 1K, 500, cull]
   - VEGETATION: [2K, 1K, 500, billboard]
2. Integrate with `lod_pipeline.py`
3. Implement silhouette validation (edge preservation check)
4. Create decimation strategy per type:
   - Characters: region-weighted vertex importance
   - Props: silhouette-preserving quadric edge collapse
   - Vegetation: billboard fallback for lowest LOD

**Success Criteria:**
- LOD presets class created
- Integrated with existing LOD pipeline
- Silhouette validation implemented
- Per-type decimation strategies defined

### Plan 1.5: Boolean Cleanup Pipeline (MESH-01.5)

**Goal:** Implement automatic cleanup after boolean operations.

**Actions:**
1. Create post-boolean cleanup function in `_mesh_bridge.py`:
   - Remove doubles with 0.0001 merge distance
   - Recalculate normals (weighted by face angles)
   - Fix non-manifold edges
   - Fill holes (up to 8 sides)
2. Add cleanup to boolean tool workflow
3. Create validation step: clean geometry check before export

**Success Criteria:**
- Boolean cleanup function created
- Integrated with all boolean operations
- Geometry passes validation after cleanup
- No non-manifold output

### Plan 1.6: Silhouette Validation (MESH-01.6)

**Goal:** Implement automatic silhouette preservation check.

**Actions:**
1. Create silhouette comparison system:
   - Compare LOD meshes against source (high to low)
   - Edge preservation score calculation
   - Failed edge identification
2. Add threshold-based validation (fail if silhouette loss > 15%)
3. Integrate with `blender_mesh` game_check action

**Success Criteria:**
- Silhouette comparison system created
- Threshold validation implemented
- Integrated with game_check action
- Automatic fail on silhouette loss

### Plan 1.7: Scene-Level Budget Validator (MESH-01.7)

**Goal:** Implement scene-level polygon budget enforcement.

**Actions:**
1. Create `SceneBudgetValidator` class:
   - Sum all visible objects in scene
   - Compare against frame budget (2M-6M tris at 60fps PC)
   - Per-room budget: 50K-150K for interiors
   - Per-block budget: 200K-500K for town blocks
2. Integrate with world composer
3. Create budget warning system (alert on exceed)
4. Add optimization suggestions (LOD culling, material consolidation)

**Success Criteria:**
- Scene budget validator created
- Per-room and per-block budgets defined
- Warning system implemented
- Optimization suggestions generated
