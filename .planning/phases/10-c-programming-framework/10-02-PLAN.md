**(SHDR-01) `generate_arbitrary_shader()`**:

```python
def generate_arbitrary_shader(
    shader_name: str,
    shader_path: str = "VeilBreakers/Custom",
    render_type: str = "Opaque",  # Opaque | Transparent | TransparentCutout
    queue: str = "",  # auto-derived from render_type if empty
    properties: list[dict] | None = None,  # [{name, display_name, type, default}]
    # type: "Float" | "Range(min,max)" | "Color" | "Vector" | "2D" | "3D" | "Cube"
    vertex_code: str = "",
    fragment_code: str = "",
    tags: dict | None = None,  # extra SubShader tags
    pragma_directives: list[str] | None = None,  # extra #pragma lines
    include_paths: list[str] | None = None,  # extra #include paths
    cull: str = "Back",  # Back | Front | Off
    zwrite: str = "",  # On | Off (auto from render_type)
    blend: str = "",  # e.g. "SrcAlpha OneMinusSrcAlpha"
    two_passes: bool = False,  # for outline-style effects
    second_pass_vertex: str = "",
    second_pass_fragment: str = "",
) -> str:
```

Build complete ShaderLab source:
1. `Shader "{shader_path}/{shader_name}"` wrapper
2. Properties block -- for each property dict, generate the appropriate ShaderLab property line. Map `type` to ShaderLab syntax:
   - `"Float"` -> `_Name ("Display", Float) = default`
   - `"Range(0,1)"` -> `_Name ("Display", Range(0, 1)) = default`
   - `"Color"` -> `_Name ("Display", Color) = (r,g,b,a)`
   - `"2D"` -> `_Name ("Display", 2D) = "white" {}`
   - `"Vector"` -> `_Name ("Display", Vector) = (x,y,z,w)`
3. SubShader with Tags -- auto-set `"RenderPipeline"="UniversalPipeline"`, `"RenderType"=render_type`, `"Queue"` from render_type
4. Pass with HLSLPROGRAM/ENDHLSL:
   - Default `#pragma vertex vert` / `#pragma fragment frag`
   - Always include `_URP_CORE_INCLUDE`
   - Variable declarations auto-generated from properties (CBUFFER_START/END with correct types)
   - Default vertex shader struct (Attributes -> Varyings with position + UV + normal + worldPos)
   - Default vertex function if vertex_code empty -- standard transform
   - Fragment function -- user code wrapped in frag function signature, or default return half4(1,1,1,1)
5. If two_passes=True, add second Pass block
6. Fallback to "Hidden/InternalErrorShader"

Sanitize shader_name for the Shader path (no quotes, no special chars).

**(SHDR-02) `generate_renderer_feature()`**:

```python
def generate_renderer_feature(
    feature_name: str,
    namespace: str = "",
    settings_fields: list[dict] | None = None,  # [{type, name, default, attribute}]
    render_pass_event: str = "BeforeRenderingPostProcessing",
    shader_property_name: str = "_shader",
    material_properties: list[dict] | None = None,  # [{name, type, value}] set on material
    pass_code: str = "",  # custom RecordRenderGraph body (if empty, generate default blit)
) -> str:
```

Generate TWO C# classes in ONE file:
1. `{feature_name}Feature : ScriptableRendererFeature`:
   - `[SerializeField] private Shader {shader_property_name};`
   - `[SerializeField] private {feature_name}Settings _settings;`
   - `private Material _material;`
   - `private {feature_name}Pass _pass;`
   - `Create()`: create material from shader, instantiate pass, set renderPassEvent
   - `AddRenderPasses()`: null-check material, enqueue pass for Game cameras only
   - `Dispose(bool)`: `CoreUtils.Destroy(_material);`
   - Nested `[System.Serializable] public class {feature_name}Settings` with settings_fields

2. `{feature_name}Pass : ScriptableRenderPass`:
   - Constructor takes Material + Settings
   - `RecordRenderGraph(RenderGraph renderGraph, ContextContainer frameData)`:
     - Get `UniversalResourceData` from frameData
     - Guard: `if (resourceData.isActiveTargetBackBuffer) return;`
     - Get source texture from `resourceData.activeColorTexture`
     - Create temp texture from source descriptor
     - Set material properties from material_properties list
     - If pass_code provided, insert it. Otherwise generate default blit + copy-back
   - Do NOT implement legacy `Execute()` -- use only `RecordRenderGraph` per URP 17

Required usings for renderer feature:
```csharp
using UnityEngine;
using UnityEngine.Rendering;
using UnityEngine.Rendering.Universal;
using UnityEngine.Rendering.RenderGraphModule;
```
