```python
def _sanitize_cs_string(value: str) -> str:
    """Escape a value for safe embedding inside a C# string literal."""
    value = value.replace("\\", "\\\\")
    value = value.replace('"', '\\"')
    value = value.replace("\n", "\\n")
    value = value.replace("\r", "\\r")
    return value

def _sanitize_cs_identifier(value: str) -> str:
    """Sanitize a value for use as a C# identifier (class name, method name)."""
    return re.sub(r"[^a-zA-Z0-9_]", "", value)
```

**(CODE-01) Core class builder -- `_build_cs_class()`**:

```python
def _build_cs_class(
    class_name: str,
    class_type: str = "class",  # "class" | "static class" | "abstract class" | "struct" | "interface" | "enum"
    namespace: str = "",
    base_class: str = "",
    interfaces: list[str] | None = None,
    usings: list[str] | None = None,
    attributes: list[str] | None = None,
    fields: list[dict] | None = None,  # [{access, type, name, default, attributes, summary}]
    properties: list[dict] | None = None,  # [{access, type, name, getter, setter, backing_field, summary}]
    methods: list[dict] | None = None,  # [{access, return_type, name, params, body, attributes, summary}]
    enum_values: list[str] | None = None,
    summary: str = "",
) -> str:
```

Build complete C# source file from structured sections:
1. Using statements -- default to `["UnityEngine"]` for MonoBehaviour/SO, empty list for plain classes. Add `"UnityEditor"` if class_type indicates editor code.
2. Namespace open -- if provided, wrap in `namespace X { }` with 4-space indent
3. XML summary comment -- if summary provided, add `/// <summary>` block
4. Class attributes -- each as `[AttributeName]`
5. Class declaration -- `public {class_type} {class_name}` with inheritance chain
6. Enum values -- if class_type is "enum", join enum_values with commas
7. Fields -- each as `{access} {type} {name}` with optional `= {default}`, use `_camelCase` for private, PascalCase for public. Support `[SerializeField]` and other attributes.
8. Properties -- auto-properties with `{ get; set; }` or custom getter/setter body
9. Methods -- full method body with params, return type. Default body is empty or `throw new NotImplementedException();`
10. Close braces

```python
    usings: list[str] | None = None,

    fields: list[dict] | None = None,
    properties: list[dict] | None = None,
    methods: list[dict] | None = None,
    enum_values: list[str] | None = None,
    summary: str = "",
) -> str:
```

Maps high-level class_type to appropriate base_class and usings:
- `"MonoBehaviour"` -> base_class=`"MonoBehaviour"`, usings include `"UnityEngine"`
- `"ScriptableObject"` -> base_class=`"ScriptableObject"`, usings include `"UnityEngine"`, add `[CreateAssetMenu]` attribute
- `"class"` / `"static class"` / `"abstract class"` / `"struct"` -> pass through to `_build_cs_class` with appropriate class_type
- `"interface"` -> generate with method signatures only (no bodies)
- `"enum"` -> generate with enum_values list

Sanitize class_name through `_safe_identifier()`. All field/method names also sanitized.

**(CODE-02) Script modification -- `modify_script()`**:

```python
def modify_script(
    source: str,
    add_usings: list[str] | None = None,
    add_fields: list[dict] | None = None,
    add_properties: list[dict] | None = None,
    add_methods: list[dict] | None = None,
    add_attributes: list[dict] | None = None,  # [{target_class, attribute}]
) -> tuple[str, list[str]]:
    """Modify existing C# source by inserting code at correct locations.

    Returns (modified_source, list_of_changes_made).
    """
```

Implementation:
1. Detect indentation style: scan for leading whitespace patterns. If tabs found, use tabs. Otherwise use spaces with detected width (default 4).
2. `add_usings`: Find last `using X;` line, insert new usings after it. Skip duplicates.
3. `add_fields`: Find insertion point after last field declaration (regex: line with access modifier + type + identifier + semicolon/equals, but NOT method signatures). Insert with matched indentation.
4. `add_properties`: Insert after last property (regex: line with `{ get;` or `{ get {`).
5. `add_methods`: Insert before the last closing brace of the class body (second-to-last `}` in the file). Generate full method with proper indentation.
6. `add_attributes`: Find class declaration line matching target_class, insert `[attribute]` on the line before it.
7. Validate balanced braces after modification. If imbalanced, raise ValueError.
8. Return (modified_source, list of description strings for each change).
