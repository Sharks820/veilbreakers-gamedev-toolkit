          - TcpListener on configurable port (default 9877), background Thread for listener loop
          - ConcurrentQueue<CommandEnvelope> for thread-safe command passing
          - Static constructor: Start() + register EditorApplication.update += ProcessCommands + AssemblyReloadEvents.beforeAssemblyReload += Stop + EditorApplication.quitting += Stop
          - Start(): create background listener thread (IsBackground=true), set _running=true
          - Stop(): set _running=false, stop listener, join thread with 2s timeout
          - ListenerLoop(): TcpListener.Start(), loop while _running, check Pending() with 50ms sleep, AcceptTcpClient(), ThreadPool.QueueUserWorkItem -> HandleClient
          - HandleClient(TcpClient): read 4-byte big-endian length prefix, read JSON bytes, create CommandEnvelope with ManualResetEventSlim, enqueue, wait 300s timeout, write response with 4-byte length prefix
          - ProcessCommands(): TryDequeue one command per update tick, call VBBridgeCommands.Dispatch(requestJson), set ResponseJson, set DoneEvent
          - ReadExactly(stream, count) helper, EscapeJson(str) helper
          - CommandEnvelope inner class: RequestJson string, ResponseJson string, DoneEvent ManualResetEventSlim
          - Required usings: UnityEngine, UnityEditor, System, System.Collections.Concurrent, System.IO, System.Net, System.Net.Sockets, System.Text, System.Threading
          - Use line-based string concatenation (lines list + "\n".join) matching existing template pattern. NOT f-strings for C# code.

       b. `generate_bridge_commands_script(namespace="")` -> str:
          Generate VBBridgeCommands.cs: static class with COMMAND_HANDLERS dictionary:
          - Dictionary<string, Func<Dictionary<string, object>, Dictionary<string, object>>> HANDLERS
          - Dispatch(string requestJson): parse JSON manually (MiniJSON-style inline parser OR use built-in regex-based simple JSON parser), lookup handler, call it, serialize response
          - Include MiniJSON as a nested static class (MIT licensed, ~200 lines, standard Unity practice) for JSON parsing since JsonUtility cannot handle Dictionary<string,object>
          - Handlers:
            - "ping": returns {status: "success", result: "pong"}
            - "recompile": AssetDatabase.Refresh(ImportAssetOptions.ForceUpdate), returns {refreshed: true}

            - "exit_play_mode": EditorApplication.ExitPlaymode(), returns {is_playing: false}
            - "screenshot": ScreenCapture.CaptureScreenshot with configurable path, returns {path: string, captured: true}
            - "console_logs": Application.logMessageReceived buffer (last N log entries), returns {logs: [{message, type, stackTrace}]}
            - "read_result": read Temp/vb_result.json, returns parsed JSON
            - "get_game_objects": iterate scene roots, return hierarchy as nested dict with name/components/children
          - SerializeResponse helper: builds JSON string from status + result dict
          - Required usings: UnityEngine, UnityEditor, System, System.Collections.Generic, System.IO, System.Text
