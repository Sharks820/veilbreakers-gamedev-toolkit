    APPEND to qa_templates.py (do NOT overwrite existing generators from Plans 01/02):

    1. `generate_crash_reporting_script(dsn="", environment="development", enable_breadcrumbs=True, sample_rate=1.0, namespace="")` -> str:
       Generate a C# runtime script (VBCrashReporting.cs) that:
       - Initializes Sentry SDK in [RuntimeInitializeOnLoadMethod(RuntimeInitializeLoadType.BeforeSceneLoad)]
       - Calls SentrySdk.Init(options => { options.Dsn = dsn; options.Environment = environment; options.SampleRate = sample_rate; options.AutoSessionTracking = true; })
       - If enable_breadcrumbs: hooks Application.logMessageReceived to add SentrySdk.AddBreadcrumb for warnings/errors
       - Provides static helper methods:
         - CaptureException(Exception e) -> SentrySdk.CaptureException(e)
         - CaptureMessage(string msg, SentryLevel level) -> SentrySdk.CaptureMessage(msg, level)
         - SetTag(string key, string value) -> SentrySdk.ConfigureScope(scope => scope.SetTag(key, value))
         - SetUser(string id, string username) -> SentrySdk.ConfigureScope(scope => scope.User = new SentryUser {...})
       - Includes fallback: if DSN is empty, logs to Unity console instead of sending to Sentry
       - Required usings: UnityEngine, Sentry (conditional #if with comment about installing Sentry package via UPM)
       - Use line-based string concatenation, not f-strings for C# code

    2. `generate_analytics_script(event_names=None, flush_interval_seconds=30, max_buffer_size=100, log_file_path="Analytics/events.json", namespace="")` -> str:
       Generate a C# runtime script (VBAnalytics.cs) that:
       - Default event_names if None: ["level_start", "level_complete", "item_acquired", "enemy_killed", "player_death", "session_start", "session_end"]
       - Implements a singleton analytics manager (MonoBehaviour, DontDestroyOnLoad)
       - TrackEvent(string eventName, Dictionary<string, object> properties = null): creates event record with {eventName, timestamp (ISO 8601), sessionId (GUID), properties}
       - Buffers events in List, flushes to disk every flush_interval_seconds or when buffer hits max_buffer_size
       - FlushEvents(): writes buffered events as JSON array to log_file_path (Application.persistentDataPath / log_file_path)
       - OnApplicationQuit: auto-flush remaining events
       - Session management: generates sessionId on Awake, tracks session_start/session_end automatically
       - Generates typed convenience methods for each event_name: e.g., TrackLevelStart(int level), TrackEnemyKilled(string enemyType, int damage)
       - Required usings: UnityEngine, System, System.Collections.Generic, System.IO

    3. `generate_live_inspector_script(update_interval_frames=10, max_tracked_objects=20, namespace="")` -> str:
       Generate a C# editor script (VBLiveInspector.cs) as an EditorWindow that:
       - Opens via [MenuItem("VeilBreakers/QA/Live Inspector")]
       - Shows live variable values from selected GameObjects during Play Mode
       - Uses EditorApplication.update to poll at update_interval_frames intervals
       - Auto-detects components on selected GameObject via GetComponents<Component>()
       - Uses Reflection (System.Reflection) to enumerate public fields and properties on each component
       - Displays in IMGUI (OnGUI) with:
         - Header: current selection name, active state, layer
         - Scrollable list of components
         - For each component: foldout with field name, type, current value
         - Special formatting: Vector3 with x/y/z, Color with swatch, bool with toggle
         - Search/filter field at top to filter by field name
       - Tracks up to max_tracked_objects via a pinned list (user can pin objects for comparison)
       - Shows FSM/state machine current state if component has a "currentState" or "_state" field
       - Shows warning banner when not in Play Mode: "Enter Play Mode to inspect live state"
       - Required usings: UnityEngine, UnityEditor, System, System.Collections.Generic, System.Reflection, System.Linq
       - Use IMGUI OnGUI pattern (consistent with existing VeilBreakers editor windows)
