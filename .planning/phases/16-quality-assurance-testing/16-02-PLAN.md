    1. `generate_memory_leak_script(growth_threshold_mb=10, sample_interval_seconds=5, sample_count=10, namespace="")` -> str:
       Generate a C# editor script (VBMemoryLeakDetector.cs) that:
       - Uses ProfilerRecorder for "GC Reserved Memory" and "System Used Memory"
       - Takes a baseline snapshot (managed_baseline_mb, native_baseline_mb)
       - Samples at intervals (sample_interval_seconds) for sample_count readings
       - Records each sample: {timestamp, managed_mb, native_mb, delta_managed, delta_native}
       - After all samples, computes: total growth (managed + native), growth rate per second, peak memory
       - Flags leak if total growth > growth_threshold_mb
       - Writes results to Temp/vb_memory_results.json
       - Results format: {baseline: {managed_mb, native_mb}, final: {managed_mb, native_mb}, growth: {managed_mb, native_mb, total_mb}, growth_rate_per_second_mb, leak_detected: bool, threshold_mb, samples: [...]}
       - Registers as [MenuItem("VeilBreakers/QA/Detect Memory Leaks")]
       - Required usings: Unity.Profiling, UnityEngine, UnityEditor, System.Collections, System.Collections.Generic, System.IO
       - Runs during Play Mode only (check EditorApplication.isPlaying, warn if not)

    2. `analyze_csharp_static(source_code: str, file_path: str = "<unknown>")` -> dict:
       This is a PYTHON function (not a C# template generator). It performs regex-based static analysis on C# source code.
       - Define ANTI_PATTERNS dict at module level with these rules:
         - "camera_main_in_update": pattern=r"Camera\.main", context=r"void\s+(Update|FixedUpdate|LateUpdate)\s*\(", severity="warning", message="Camera.main in Update loop -- cache in Awake/Start", fix suggestion
         - "getcomponent_in_update": pattern=r"GetComponent<\w+>\(\)", context same Update methods, severity="warning"
         - "find_object_at_runtime": pattern=r"FindObjectOfType|FindObjectsOfType|FindFirstObjectByType", context=r"void\s+(Update|FixedUpdate|LateUpdate|OnTrigger|OnCollision)", severity="error"
         - "string_concat_in_update": pattern=r'\+\s*"[^"]*"|\"\s*\+', context Update methods, severity="info"
         - "linq_in_update": pattern=r"\.(Where|Select|Any|All|First|Last|Count|OrderBy|GroupBy|ToList|ToArray)\s*\(", context Update methods, severity="warning"
         - "new_allocation_in_update": pattern=r"\bnew\s+\w+[\[\(]", context Update methods, severity="warning", message="Allocation in Update loop -- consider object pooling"
       - Algorithm:
         a. Split source_code into lines
         b. Track which method body each line belongs to (simple brace counter: when encountering void Update/FixedUpdate/LateUpdate, set in_hot_method=True, count braces)
         c. For each ANTI_PATTERN, check if pattern matches within a hot method body
         d. Collect findings: {rule_name, severity, line_number, line_content, message, fix}
       - Return dict: {file_path, findings_count, findings: [...], summary: {errors, warnings, infos}}
       - Handle edge cases: empty source, no methods, no findings (clean report)
