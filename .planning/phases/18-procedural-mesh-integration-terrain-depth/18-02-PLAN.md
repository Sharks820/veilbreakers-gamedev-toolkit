Create `Tools/mcp-toolkit/blender_addon/handlers/_terrain_depth.py` as a pure-logic module (NO bpy/bmesh imports). Import _make_result and _merge_meshes from `.procedural_meshes`. Follow the exact same MeshSpec return format.

**Generator 1: generate_cliff_face_mesh()**
Parameters: width=20.0, height=15.0, segments_horizontal=16, segments_vertical=12, noise_amplitude=0.8, noise_scale=3.0, seed=0, style="granite"
Algorithm: Create a curved vertical surface (partial cylinder segment). For each (ix, iy) grid point, compute position as: x = (ix/seg_h - 0.5) * width, y = iy/seg_v * height (Y is vertical), z = base_curve + noise. Base curve = 0.3 * sin(x_frac * pi) for natural concavity. Noise via random.Random(seed).gauss(). Generate quad faces for each grid cell. Return via _make_result with category="terrain_depth". Per user decision: noise-displaced cylinder segments, NOT heightmap-based.

**Generator 2: generate_cave_entrance_mesh()**
Parameters: width=4.0, height=4.0, depth=3.0, arch_segments=12, terrain_edge_height=0.0, style="natural", seed=0
Algorithm: Generate an archway frame (half-cylinder top + rectangular sides) with irregular surface displacement. The arch profile is a semicircle of radius=width/2 at height. Add side walls extending into the terrain (depth). Bottom edge vertices match terrain_edge_height for seamless placement. Add interior tunnel segment. Return via _make_result with category="terrain_depth".

**Generator 3: generate_biome_transition_mesh()**
Parameters: biome_a="forest", biome_b="swamp", zone_width=10.0, zone_depth=20.0, segments=12, seed=0
Algorithm: Generate a ground-level mesh strip representing the transition zone. Vertices have noise-displaced height to create uneven ground. The mesh is subdivided into grid cells. Metadata includes biome_a, biome_b names for material assignment. Vertex colors (stored in metadata as vertex_groups) indicate blend weight from biome_a (0.0) to biome_b (1.0) across the width. Return via _make_result with category="terrain_depth".

**Generator 4: generate_waterfall_mesh()**
Parameters: width=3.0, height=10.0, steps=4, step_depth=0.5, pool_radius=2.0, style="rocky_cascade", seed=0
Algorithm: Generate stepped cascade geometry. Each step is a horizontal water surface plane at a decreasing height, connected by vertical water curtain faces. Steps have slight random width variation via seed. At the base, generate a circular pool mesh (disk). The cascade faces get slight noise displacement for non-uniform water surface. Return via _make_result with category="terrain_depth".

**Generator 5: generate_terrain_bridge_mesh()**
Parameters: start_pos=(0,0,0), end_pos=(10,0,0), width=3.0, style="stone_arch", railing=True, arch_height=2.0, segments=12, seed=0
Algorithm: Wrapper around procedural_meshes.generate_bridge_mesh() that adds terrain-aware placement. Compute span from distance between start_pos and end_pos. Compute rotation angle from direction vector. Call generate_bridge_mesh(span, width, arch_height, style, railing, segments). Transform all vertices by rotating to match the direction and offsetting to start_pos. Return the transformed MeshSpec. For rope style: use drooping catenary curve instead of arch. For drawbridge style: flat plank surface with chain attachment points.
