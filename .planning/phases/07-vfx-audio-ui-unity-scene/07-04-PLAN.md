    UI Templates (ui_templates.py):
    - generate_uxml_screen(spec) takes dict with title, elements list (label/button/image/panel/input/slider/toggle), returns valid UXML string with xmlns declaration
    - generate_uss_stylesheet(theme="dark_fantasy") returns USS string with VeilBreakers dark fantasy styling: dark backgrounds (#1a1a2e), glowing accent colors (#4a0e4e, #e6e6ff), hover states
    - generate_responsive_test_script(uxml_path, resolutions) returns C# editor script that sets GameView to each resolution, captures screenshots, saves as {uxml_name}_{res}.png
    - validate_uxml_layout(uxml_string) parses UXML XML, returns dict of issues: overlaps (bounding box intersection), zero-size elements (width=0 or height=0), overflow (child > parent). Pure string/XML parsing, no Unity needed.

    WCAG Checker (wcag_checker.py):
    - relative_luminance(r, g, b) -- W3C formula, RGB values 0-1
    - contrast_ratio(fg_rgb, bg_rgb) -- both tuples (0-255), returns float ratio
    - check_wcag_aa(fg, bg, large_text=False) -- returns True if >= 4.5:1 (3:1 for large text)
    - validate_uxml_contrast(uxml_string, uss_string) -- extracts text colors and background colors from USS, checks all text elements against WCAG AA, returns list of violations with element name, fg, bg, ratio

    Screenshot Diff (screenshot_diff.py):
    - compare_screenshots(reference_path, current_path, threshold=0.01) -> dict with {"match": bool, "diff_percentage": float, "diff_image_path": str | None}
    - generate_diff_image(reference_path, current_path, output_path) -> str -- saves highlighted diff image, returns output_path
    - Uses Pillow ImageChops.difference() for pixel comparison
    - Threshold is max acceptable percentage of changed pixels (0.01 = 1%)
  </behavior>

    **ui_templates.py**:
    - `generate_uxml_screen(spec: dict) -> str` -- spec has: `{"title": str, "elements": [{"type": "label|button|image|panel|input|slider|toggle", "text": str, "name": str, "class": str, "children": [...]}]}`. Recursively builds UXML tree. Root element always has `class="screen-root"`.
    - `generate_uss_stylesheet(theme: str = "dark_fantasy") -> str` -- Returns complete USS with classes: `.screen-root`, `.vb-title`, `.vb-button`, `.vb-button:hover`, `.vb-label`, `.vb-panel`, `.vb-input`, `.vb-slider`, `.vb-toggle`. Dark fantasy defaults: bg #1a1a2e, text #e6e6ff, accent #4a0e4e, hover glow #7b2d8e.
    - `generate_responsive_test_script(uxml_path: str, resolutions: list[tuple[int,int]] | None = None) -> str` -- C# editor script that for each resolution: (1) sets GameView size via internal GameView API using reflection, (2) waits one frame, (3) captures screenshot to Assets/Screenshots/{name}_{w}x{h}.png. Default 5 resolutions: [(1280,720), (1920,1080), (2560,1440), (3840,2160), (800,600)].
    - `validate_uxml_layout(uxml_string: str) -> dict` -- Parses UXML as XML (xml.etree.ElementTree). Checks for: elements with explicit width=0 or height=0 in style attributes, nested elements that would overflow parent (based on explicit size values when present), duplicate names. Returns `{"valid": bool, "issues": [{"type": "zero_size|overflow|duplicate_name", "element": str, "details": str}]}`.

    **wcag_checker.py**:
    - Implement exact W3C relative luminance formula with sRGB linearization
    - `validate_uxml_contrast` parses USS to extract color properties, maps elements between UXML and USS by class/name, computes contrast for each text element against its background. Uses regex to parse `color:` and `background-color:` from USS.

    **screenshot_diff.py**:
    - `compare_screenshots` opens both images with Pillow, resizes current to match reference if different sizes, computes difference, counts changed pixels (above small noise threshold of 10/255 per channel), returns match/diff_percentage
    - `generate_diff_image` creates a red-highlighted overlay showing changed regions

    ```python
    @mcp.tool()
    async def unity_ui(
        action: Literal[
            "generate_ui_screen",      # UI-05: UXML + USS from description
            "validate_layout",         # UI-02: check overlaps, zero-size, overflow
            "test_responsive",         # UI-03: capture at 5 resolutions
            "check_contrast",          # UI-06: WCAG AA contrast validation
            "compare_screenshots",     # UI-07: visual regression detection
        ],
        # Screen generation params
        screen_spec: dict | None = None,
        theme: str = "dark_fantasy",
        screen_name: str = "default",
        # Validation params
        uxml_path: str = "",
        uss_path: str = "",
        uxml_content: str = "",
        uss_content: str = "",
        # Responsive test params
        resolutions: list[list[int]] | None = None,
        # Screenshot comparison params
        reference_path: str = "",
        current_path: str = "",
        diff_threshold: float = 0.01,
    ) -> str:
    ```

    Actions:
    1. `generate_ui_screen`: Calls generate_uxml_screen(screen_spec) + generate_uss_stylesheet(theme), writes UXML to `Assets/Resources/UI/{screen_name}.uxml` and USS to `Assets/Resources/UI/{screen_name}.uss`.     3. `test_responsive`: Generates responsive test C# script, writes to `Assets/Editor/Generated/UI/ResponsiveTest_{screen_name}.cs`, returns file_path + menu_item.

    5. `compare_screenshots`: Calls compare_screenshots + generate_diff_image, returns match/diff_percentage/diff_image_path.
