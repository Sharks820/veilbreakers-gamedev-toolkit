  <action>
    Create `shared/unity_templates/` package with `__init__.py` (empty) and `editor_templates.py`.

    In editor_templates.py, implement 5 pure functions that return C# source code strings:
    1. `generate_recompile_script()` -- C# editor script that calls AssetDatabase.Refresh(ImportAssetOptions.ForceUpdate), writes JSON result to `Temp/vb_result.json`
    2. `generate_play_mode_script(enter: bool = True)` -- C# that calls EditorApplication.EnterPlaymode() or ExitPlaymode(), writes success JSON
    3. `generate_screenshot_script(output_path: str = "Screenshots/vb_capture.png", supersize: int = 1)` -- C# that captures game view screenshot via ScreenCapture.CaptureScreenshot, writes path to result JSON
    4. `generate_console_log_script(filter_type: str = "all", count: int = 50)` -- C# that uses Application.logMessageReceived or LogEntries internal API to collect recent log entries, writes them as JSON array to result file
    5. `generate_gemini_review_script(screenshot_path: str, criteria: list[str])` -- C# editor script that reads screenshot, but actual Gemini call is Python-side (this just exports the screenshot path)

    Create `shared/gemini_client.py` with GeminiReviewClient class:
    - Constructor takes optional api_key (falls back to env var GEMINI_API_KEY)
    - `review_screenshot(image_path: str, prompt: str) -> dict` -- sends image to Gemini API for visual quality review. Returns {"quality_score": float, "issues": list, "suggestions": list, "summary": str}
    - Stub mode when no API key: returns placeholder dict with quality_score=0.0 and summary="Gemini review unavailable (no API key)"
    - Uses httpx or google-generativeai SDK for API call

    Every generated C# script follows the pattern:
    - `using UnityEngine; using UnityEditor;`
    - `public static class VeilBreakers_{Name} { [MenuItem("VeilBreakers/Editor/{Action}")] public static void Execute() { ... } }`
    - Writes result to `Temp/vb_result.json` so Python can read it back after execution

    Write tests FIRST (RED), then implement (GREEN). Tests verify:
    - Each function returns a string containing expected C# keywords (using statements, class names, API calls)
    - Parameter substitution works (paths, filter types, resolution values appear in output)
    - Invalid inputs raise ValueError (e.g., supersize < 1)
    - Gemini client stub mode returns correct structure
