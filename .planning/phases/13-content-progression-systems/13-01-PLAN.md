Generate these functions:

1. **generate_inventory_system_script(grid_width=8, grid_height=5, equipment_slots=None, namespace="VeilBreakers.Content")** -> tuple[str, str, str, str]
   Returns (item_so_cs, inventory_cs, uxml, uss).
   - item_so_cs: VB_ItemData SO class with ItemType enum (Consumable=0, Weapon=1, Armor=2, Accessory=3, Material=4, KeyItem=5), ItemRarity enum (Common=0 through Legendary=4), stat_buffs array, corruptionChange, pathChange. Must have [CreateAssetMenu]. Must match items.json schema exactly.
   - inventory_cs: VB_InventorySystem MonoBehaviour with grid slots, equipment slots (default: Head, Torso, Arms, Legs, Weapon, Shield, Accessory1, Accessory2), Add/Remove/Equip/Unequip methods, storage container support. Uses EventBus for cross-system notifications.
   - uxml: Grid-based inventory layout with equipment slots panel
   - uss: Dark fantasy themed styling

2. **generate_dialogue_system_script(namespace="VeilBreakers.Content")** -> tuple[str, str, str, str]
   Returns (dialogue_data_cs, dialogue_system_cs, uxml, uss).
   - dialogue_data_cs: VB_DialogueNode SO with speaker, lines, choices (text + condition + next_node), YarnSpinner-compatible node generation (title/---/===/-> choices/<<commands>>)
   - dialogue_system_cs: VB_DialogueSystem MonoBehaviour with Start/AdvanceLine/SelectChoice/End, condition checking (quest state, inventory, reputation). NPC interaction trigger extending interactable pattern from Phase 12.
   - uxml: Speaker portrait, text area, choice buttons
   - uss: Dark fantasy dialogue styling

3. **generate_quest_system_script(namespace="VeilBreakers.Content")** -> tuple[str, str, str, str]
   Returns (quest_data_cs, quest_system_cs, uxml, uss).
   - quest_data_cs: VB_QuestData SO with objectives list (Kill/Collect/TalkTo/ReachLocation), rewards (XP, gold, items), quest giver NPC reference
   - quest_system_cs: VB_QuestSystem with state machine (NotStarted/Active/Complete/TurnedIn), objective tracking with event-driven completion, reward distribution to inventory/XP/currency via EventBus
   - uxml: Quest log with main/side/daily tabs, tracking markers
   - uss: Dark fantasy quest log styling

4. **generate_loot_table_script(namespace="VeilBreakers.Content")** -> str
   Returns loot_table_cs.
   - VB_LootTable SO with LootEntry (item ref, weight, minRarity, quantity range, brandAffinity)
   - Roll(Brand monsterBrand, float corruptionLevel) using weighted random with cumulative distribution
   - VB-08: Brand affinity bonus -- delegates to BrandSystem.GetEffectiveness(), boosts matching brand gear weight by 1.5x
   - Corruption bonus: corruptionLevel > 0.5 multiplies weight by (1 + corruption * 0.3)

5. **generate_crafting_system_script(namespace="VeilBreakers.Content")** -> tuple[str, str]
   Returns (recipe_cs, crafting_cs).
   - recipe_cs: VB_Recipe SO with ingredient list (item + count), result item, crafting station type requirement, unlock conditions (quest/level)
   - crafting_cs: VB_CraftingSystem MonoBehaviour with CanCraft/Craft methods, station interaction, ingredient validation against inventory, unlock progression tracking

6. **generate_skill_tree_script(hero_paths=None, namespace="VeilBreakers.Content")** -> tuple[str, str]
   Returns (skill_node_cs, skill_tree_cs).
   - skill_node_cs: VB_SkillNode SO with prerequisites list (other nodes), brand requirement, stat bonuses, unlock cost (skill points)
   - skill_tree_cs: VB_SkillTree MonoBehaviour with hero path layouts (default IRONBOUND/FANGBORN/VOIDTOUCHED/UNCHAINED), AllocatePoint/ResetPoints, prerequisite validation, available points tracking per level

7. **generate_dps_calculator_script(brands=None, namespace="VeilBreakers.Editor.Balancing")** -> str
   Returns dps_calculator_cs. EDITOR ONLY -- must have `using UnityEditor;`.
   - VB_DPSCalculator EditorWindow with input fields for base ATK, weapon damage, attack speed, crit chance, crit multiplier
   - Brand/synergy modifier dropdowns (10 brands)
   - Calculate DPS = (baseDmg * atkSpeed) * (1 + critChance * (critMult - 1)) * brandMultiplier * synergyMultiplier
   - OnGUI layout with results display

8. **generate_encounter_simulator_script(namespace="VeilBreakers.Editor.Balancing")** -> str
   Returns encounter_sim_cs. EDITOR ONLY -- must have `using UnityEditor;`.
   - VB_EncounterSimulator EditorWindow running N Monte Carlo encounters
   - Inputs: player stats, enemy stats, encounter count
   - Outputs: win rate, average duration, average damage taken, DPS distribution
   - OnGUI with simulation results and run button

9. **generate_stat_curve_editor_script(namespace="VeilBreakers.Editor.Balancing")** -> str
   Returns stat_curve_cs. EDITOR ONLY -- must have `using UnityEditor;`.
   - VB_StatCurveEditor EditorWindow with AnimationCurve fields for HP/ATK/DEF scaling per level
   - Enemy type dropdown, level range slider
   - Visual curve preview using EditorGUILayout.CurveField
   - Export to JSON button for balance data

10. **generate_shop_system_script(namespace="VeilBreakers.Content")** -> tuple[str, str, str, str]
    Returns (merchant_data_cs, shop_system_cs, uxml, uss).
    - merchant_data_cs: VB_MerchantInventory SO with item list, price overrides, restock timer
    - shop_system_cs: VB_ShopSystem MonoBehaviour with Buy/Sell, price display with currency formatting, equipment stat comparison (compare equipped vs shop item), currency transaction via EventBus
    - uxml: Shop UI with item grid, price, stat comparison panel, buy/sell buttons
    - uss: Dark fantasy shop styling

11. **generate_journal_system_script(namespace="VeilBreakers.Content")** -> tuple[str, str, str, str]
    Returns (journal_data_cs, journal_system_cs, uxml, uss).
    - journal_data_cs: VB_JournalEntry SO with entry type (Lore/Bestiary/Item), title, content, discovery condition, monster stats/weaknesses (for bestiary)
    - journal_system_cs: VB_JournalSystem MonoBehaviour with DiscoverEntry/GetEntries(type), progressive unlock, EventBus integration for discovery events
    - uxml: Tabbed layout (Lore/Bestiary/Items), entry list with detail pane
    - uss: Dark fantasy journal styling

CRITICAL RULES:
- ItemType enum values MUST be: Consumable=0, Weapon=1, Armor=2, Accessory=3, Material=4, KeyItem=5 (matching items.json)
- ItemRarity enum values MUST be: Common=0, Uncommon=1, Rare=2, Epic=3, Legendary=4 (matching items.json)
- Only DPS calculator, encounter simulator, and stat curve editor use `using UnityEditor;`. ALL other generators are runtime-only.
- Brand loot affinity (VB-08) MUST delegate to BrandSystem.GetEffectiveness(), never reimplement.
- Use EventBus pattern for cross-system communication (quest completion -> inventory reward, etc.)
- All public C# classes must be prefixed with VB_ per VeilBreakers convention.
