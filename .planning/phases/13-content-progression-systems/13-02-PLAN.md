Create equipment.py in handlers/ with 4 async handler functions following the established handler pattern (async def, params dict input, dict return):

1. **equipment_generate_weapon(params: dict) -> dict** (EQUIP-01)
   Params: weapon_type (str: sword/axe/mace/staff/bow/dagger/shield), length (float), blade_width/head_size (float), material_name (str, optional)

   Generates parametric weapon mesh using bmesh:
   - **Sword**: Tapered blade with cross-guard, hilt, pommel. Blade is a subdivided quad strip tapering to point.
   - **Axe**: Handle cylinder + axe head wedge shape with edge loop at cutting edge.
   - **Mace**: Handle cylinder + sphere/studded head.
   - **Staff**: Long cylinder with slight taper, ornamental top (sphere or crystal placeholder).
   - **Bow**: Curved limb arc (bezier-like vertex placement), string line between endpoints.
   - **Dagger**: Short sword variant -- shorter blade, wider proportionally.
   - **Shield**: Rounded rectangle or kite-shape flat mesh with boss (center bump).

   For ALL weapon types, create these child empties:
   - `grip_point`: At hilt/handle center (where hand grabs)
   - `trail_attach_top`: At weapon tip/striking end (for VFX trail)
   - `trail_attach_bottom`: At blade base / just above guard (trail start)
   - `collision_mesh`: Simplified convex hull of weapon shape as separate mesh object

   Auto-applies smooth shading to weapon mesh. Returns: {"object_name", "weapon_type", "vertices", "faces", "empties": ["grip_point", "trail_attach_top", "trail_attach_bottom"], "collision_object_name"}.

2. **equipment_split_character(params: dict) -> dict** (EQUIP-03)
   Params: object_name (str), parts (list of str, default ["head", "torso", "upper_arms", "lower_arms", "upper_legs", "lower_legs", "feet"])

   Splits a rigged character mesh into modular parts by vertex groups:
   - Verify mesh has an armature modifier and vertex groups matching part names
   - For each part: select vertices belonging to that vertex group (weight > 0.5), separate by selection
   - Each separated part retains the SAME armature modifier pointing to the shared skeleton
   - Name separated objects as `{original_name}_{part}` (e.g., "Character_head", "Character_torso")
   - Create seam edges at part boundaries for clean UV splits

   Returns: {"original_name", "parts": [{"name": "Character_head", "vertices": N, "faces": N}, ...], "armature_name"}.

3. **equipment_fit_armor(params: dict) -> dict** (EQUIP-04)
   Params: armor_object_name (str), character_object_name (str), use_shape_keys (bool, default True), body_types (list of str, default ["default", "muscular", "slim"])

   Fits armor mesh to character body:
   - Add Surface Deform modifier to armor mesh, bind to character mesh
   - Transfer vertex weights from character to armor using Data Transfer modifier (nearest face interpolated)
   - If use_shape_keys: for each body type, apply shape key to character, apply surface deform on armor, save as shape key on armor
   - Assign same armature as character to armor mesh
   - Apply Surface Deform modifier (keep as shape keys were captured)

   Returns: {"armor_name", "character_name", "shape_keys": ["default", "muscular", "slim"], "weight_groups_transferred": N, "armature_name"}.

4. **equipment_render_icon(params: dict) -> dict** (EQUIP-05)
   Params: object_name (str), output_path (str), resolution (int, default 256), camera_distance (float, default 2.0), camera_angle (tuple, default (30, 45, 0) for pitch/yaw/roll), background_alpha (float, default 0.0 for transparent)

   Renders equipment preview icon:
   - Create temporary camera at calculated position from camera_distance + camera_angle
   - Create 3-point studio lighting setup (key light, fill light, rim light)
   - Set render resolution to resolution x resolution
   - Set film transparent = True for transparent background
   - Render to output_path as PNG with alpha channel
   - Clean up temporary camera and lights

   Returns: {"object_name", "output_path", "resolution", "file_size_bytes"}.
