**Part A: Create equipment_templates.py (EQUIP-06)**

Create equipment_templates.py with a single generator following game_templates.py pattern:

**generate_equipment_attachment_script(namespace="VeilBreakers.Content")** -> tuple[str, str]
Returns (attachment_cs, weapon_sheath_cs).

attachment_cs (VB_EquipmentAttachment MonoBehaviour):
- SkinnedMeshRenderer bone rebinding using name-based Dictionary<string, Transform> lookup (NOT index-based)
- BuildBoneMap(Transform root) recursive helper
- RebindToArmature(SkinnedMeshRenderer equipmentSMR, Transform armatureRoot) method
- EquipArmor(SkinnedMeshRenderer armorSMR) / UnequipArmor(string slotName) methods
- Equipment slot tracking (dictionary of slot -> equipped SMR)
- EventBus integration for equipment change events
- References Phase 9 bone socket system (10 standard sockets: weapon_hand_R/L, shield_hand_L, back_weapon, hip_L/R, head, chest, spell_hand_R/L)

weapon_sheath_cs (VB_WeaponSheath MonoBehaviour):
- Uses Animation Rigging Multi-Parent Constraint for weighted blending between drawn/sheathed positions
- SetDrawn()/SetSheathed() methods that animate constraint source weights
- Configurable drawn and sheathed parent transforms (bone sockets)
- Requires com.unity.animation.rigging package
- Uses `using UnityEngine.Animations.Rigging;`

CRITICAL: These are RUNTIME scripts. No `using UnityEditor;`.
